## Findings Closed During Audit

The initial safe-fixture mutation uncovered a selected-report lookup bug:

- Artifact edit draft child entities included `parent_id`, `student_id`, and `week_start`, so they appeared in `GSI-ParentId`.
- `get_report_for_child_by_week` could return a child entity instead of the `SK=SUMMARY` report row.
- Artifact edit apply then misclassified the report as stale and returned 409.

Fix:

- Commit `123faad299d0fc6051b7677c8b75cb96df63c9e3` filters parent report lookups to `SK == SUMMARY` and defensively skips non-summary rows.
- Regression coverage added in `tests/test_parent_children.py`.
- Backend deploy `27070767161` completed successfully.
- Safe-fixture mutation/cleanup smoke passed after deploy.

## Residual Risks

- The synthetic safe fixture remains in production for future controlled verification. Operators should keep treating it as the only approved artifact mutation target unless another named fixture is explicitly provisioned.
- The safe-fixture harness currently restores the report metadata pointer but does not delete the intermediate versioned artifact objects. This is acceptable for version-history verification, but long-term fixture retention policy can be defined later.
- Rich version selection/history browsing remains out of scope.
