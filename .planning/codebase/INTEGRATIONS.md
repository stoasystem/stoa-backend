# Integrations

**Mapped:** 2026-06-02
**Scope:** Full repository

## Summary

The backend integrates primarily with AWS services: Cognito, DynamoDB, S3, Bedrock, Rekognition, SQS, SES, Lambda, and API Gateway. It also uses HTTPX to fetch Cognito JWKS for token validation.

## AWS Cognito

Authentication and user identity are implemented with Cognito.

Relevant files:

- `src/stoa/config.py`
- `src/stoa/deps.py`
- `src/stoa/routers/auth.py`
- `src/stoa/routers/students.py`

Key behaviors:

- `src/stoa/routers/auth.py` creates users with `admin_create_user`, sets permanent passwords with `admin_set_user_password`, adds users to role groups, and authenticates with `USER_PASSWORD_AUTH`.
- Role-specific client IDs are selected from settings by `_client_id_for_role`.
- Frontend role `tutor` maps to backend role `teacher`.
- `src/stoa/deps.py` validates Cognito access tokens using JWKS fetched from the configured user pool.
- `src/stoa/deps.py` resolves roles from `cognito:groups`, falls back to `custom:role`, then tries a DynamoDB/Cognito admin lookup.
- `src/stoa/routers/auth.py` and `src/stoa/routers/students.py` call `admin_get_user` to resolve email/profile details from Cognito usernames.

## DynamoDB

The backend uses a single DynamoDB table, configured as `settings.dynamodb_table_name`.

Relevant files:

- `src/stoa/db/dynamodb.py`
- `src/stoa/db/repositories/user_repo.py`
- `src/stoa/db/repositories/question_repo.py`
- `src/stoa/db/repositories/practice_repo.py`
- `src/stoa/db/repositories/report_repo.py`
- Direct table access also appears in `src/stoa/routers/admin.py`, `src/stoa/routers/conversations.py`, `src/stoa/routers/parents.py`, `src/stoa/routers/students.py`, `src/stoa/routers/teachers.py`, and `src/stoa/routers/tutors.py`.

Observed partition patterns:

- Users: `PK=USER#{user_id}`, `SK=PROFILE`.
- Questions: `PK=QUESTION#{question_id}`, `SK=META`.
- Reports: `PK=REPORT#{report_id}`, `SK=SUMMARY`.
- Practice content: `PK=PRACTICE`, with `SK` prefixes `SUBJECT#`, `TOPIC#`, `UNIT#`, `LESSON#`, and `CHALLENGE#`.
- Student progress: `PK=PROGRESS#{user_id}`, `SK=LESSON#{lesson_id}`.
- Mistakes: `PK=MISTAKES#{user_id}`, `SK=ATTEMPT#{uuid}`.
- Usage counters: `PK=USAGE#{student_id}`, `SK=CHAT#{date}` or `HINT#{date}`.
- Conversations: `PK=CONV#{conversation_id}`, `SK=CONV`, `MSG#{message_id}`, or `NOTE#{note_id}`.
- Teacher sessions: `PK=SESSION#{session_id}`, `SK=META`.

Observed indexes:

- `GSI-Email` is used by `src/stoa/db/repositories/user_repo.py`.
- `GSI-StudentId` is used by `src/stoa/db/repositories/question_repo.py` and `src/stoa/routers/conversations.py`.
- `GSI-ParentId` is used by `src/stoa/db/repositories/report_repo.py`.

## Amazon S3

S3 is used for direct client image uploads and OCR input.

Relevant files:

- `src/stoa/routers/files.py`
- `src/stoa/services/ocr_service.py`
- `src/stoa/config.py`

Key behavior:

- `POST /files/presign` creates presigned PUT URLs for image uploads.
- Allowed extensions are `jpg`, `jpeg`, `png`, `gif`, `webp`, and `heic`.
- Content type must start with `image/`.
- Upload keys are generated as `uploads/{user_id}/{uuid}.{ext}`.
- `src/stoa/services/ocr_service.py` passes S3 bucket/key references to Rekognition.

## Amazon Bedrock

Bedrock is used for AI tutoring, hints, and conversation title generation.

Relevant files:

- `src/stoa/services/ai_service.py`
- `src/stoa/routers/conversations.py`
- `src/stoa/routers/practice.py`

Key behavior:

- `src/stoa/services/ai_service.py` invokes `bedrock-runtime` with Anthropic message payloads.
- The main tutor answer path asks for strict JSON output, parses and validates it, and redacts forbidden internal terms.
- The conversation flow includes recent message history and formats model output as student-visible text.
- Practice hints can use a shorter plain-text Bedrock call when a challenge has no static hint.
- Conversation titles are generated by Bedrock in `_generate_title` in `src/stoa/routers/conversations.py`.

## Amazon Rekognition

Rekognition is used for text extraction from student-uploaded homework images.

Relevant files:

- `src/stoa/services/ocr_service.py`
- `src/stoa/routers/questions.py`

Key behavior:

- `detect_text` is called with an S3 object reference.
- Only line-level detections are returned.
- Lines are sorted by top coordinate before being joined.
- OCR failure in question submission is non-fatal.

## Amazon SQS

SQS is used to enqueue human teacher escalation from the question flow.

Relevant files:

- `src/stoa/services/notify_service.py`
- `src/stoa/routers/questions.py`

Key behavior:

- `enqueue_teacher_request` sends a message to `settings.teacher_queue_url`.
- It assumes a FIFO queue and sets `MessageGroupId=subject` and `MessageDeduplicationId=question_id`.

## Amazon SES

SES is used for weekly report email delivery.

Relevant file:

- `src/stoa/services/notify_service.py`

Key behavior:

- `send_weekly_report_email` sends HTML reports from `noreply@stoa.ch`.

## AWS Lambda and API Gateway

Production runtime is AWS Lambda behind API Gateway HTTP API.

Relevant files:

- `src/stoa/main.py`
- `.github/workflows/deploy.yml`

Key behavior:

- Mangum wraps the FastAPI app with `lifespan="off"`.
- GitHub Actions updates the Lambda function named `stoa-api`.
- Streaming responses are documented as buffered by API Gateway in `src/stoa/routers/conversations.py`.

## External HTTP

`src/stoa/deps.py` uses `httpx.get` to fetch Cognito JWKS with a five-second timeout and caches keys for the Lambda cold start lifecycle.
