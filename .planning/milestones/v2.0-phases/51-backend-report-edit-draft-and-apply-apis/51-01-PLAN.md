# Phase 51 Plan

## Scope

Add admin-only backend APIs for controlled report edit drafts:

- `POST /admin/reports/{parent_id}/{student_id}/{week_start}/edit-drafts`
- `GET /admin/reports/{parent_id}/{student_id}/{week_start}/edit-drafts/{draft_id}`
- `POST /admin/reports/{parent_id}/{student_id}/{week_start}/edit-drafts/{draft_id}/apply`

## Work Items

1. Add repository helpers for draft persist/read/apply marker and source-timestamp guarded report update.
2. Add `report_edit_service` with allowlist validation, stale draft rejection, privacy marker rejection, and audit writes.
3. Add admin router models/routes and safe error mapping.
4. Add focused tests for admin-only access, metadata-only responses, denylist validation, apply mutation, stale rejection, and audit evidence.
5. Run focused and full backend verification.

## Safety Gates

- No production mutation during local verification.
- Responses must omit S3 keys, presigned URLs, raw HTML, raw JSON, and token fields.
- Apply must reject drafts whose source `updated_at` no longer matches the report.
