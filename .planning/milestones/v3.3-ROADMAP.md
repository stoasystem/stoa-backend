# Roadmap: v3.3 Subscription Operations MVP

**Status:** Active
**Started:** 2026-06-08

## Goal

Make the MVP manual subscription model usable before Stripe/TWINT integration.

## Planned Phases

| Phase | Name | Status | Requirement |
|-------|------|--------|-------------|
| 100 | Subscription Operations Contract And Entitlement Model | Planned | SUBOPS-01 |
| 101 | Backend Subscription Request And Admin Tier APIs | Planned | SUBOPS-02 |
| 102 | Parent Subscription Management UI And Admin Queue | Planned | UI-18 |
| 103 | v3.3 Functional Release Gate And Billing Readiness | Planned | VERIFY-16 |

## Release Gate

v3.3 is complete when:

- Subscription operation and entitlement contract is documented.
- Backend parent request/admin processing APIs work for the manual MVP flow.
- Parent plan/request UI and admin subscription queue UI are integrated.
- Focused backend/frontend checks and browser smoke prove the workflow works.
- `STOA_DOCS_FEATURE_GAP_AUDIT.md` records manual subscription operations outcome and keeps Stripe/TWINT as future integration.
