# Requirements: v5.32 Live Pilot Remediation And Reliability Fixes

**Milestone:** v5.32
**Status:** Contract Complete
**Created:** 2026-07-07
**Prior milestone:** v5.31 Real Limited Pilot Execution Operations

## Purpose

Use live pilot evidence to fix the most important issues before expansion. v5.32 should improve activation, learning value, support resolution, mobile stability, billing/account trust, and reliability.

## Requirements

### LIVEFIX-01 Live Pilot Issue Triage And Fix Plan

Acceptance criteria:

- Live issues are grouped by severity, frequency, owner, affected role, product surface, and expansion impact.
- Must-fix expansion blockers are separated from accepted limitations and future enhancements.
- Each must-fix item has evidence source and verification plan.
- Deferred issues have user/support explanation.

### LIVEFIX-02 Account Mobile Billing Notification Fixes

Acceptance criteria:

- High-impact account, verification, entitlement, billing, mobile, offline, and notification issues are fixed or explicitly disabled.
- Fixes preserve no-demo-fallback behavior and support-safe state explanations.
- Regression tests cover reproduced issues.
- Support/admin views expose repaired state clearly.

### LIVEFIX-03 Learning AI Curriculum Teacher-Help Fixes

Acceptance criteria:

- AI quality, exercise generation, assignment relevance, curriculum placement, teacher-help routing, and parent/student explanation issues are remediated or scoped.
- Teacher oversight and safety escalation remain intact.
- Learning fixes are validated against live examples or approved synthetic fixtures.
- Raw student work is not copied into public docs or analytics evidence.

### LIVEFIX-04 Regression Release And Support Evidence

Acceptance criteria:

- Focused tests and smoke checks cover fixed live pilot issues.
- Reliability evidence covers impacted APIs, frontend/mobile flows, provider states, and support operations.
- Dashboards or support exports show issue status, reopen rate, and remaining blockers.
- Release notes explain changes in user/operator language.

### VERIFY-66 Remediation Gate

Acceptance criteria:

- Decision is expansion-ready, continue pilot, or another remediation cycle.
- Roadmap, requirements, state, milestone snapshots, and project summary are updated.
- v5.33 expansion scope is based on resolved live pilot evidence.
- Expansion remains blocked if must-fix blockers remain.

## Out of Scope

- Expansion before must-fix blockers are resolved.
- Broad redesign unrelated to live pilot evidence.
- Public launch.
- Unreviewed AI autonomy expansion.
- Provider behavior that bypasses rollout controls.
