# Roadmap: v5.36 Live Pilot Operations Feedback And Product Fixes

**Status:** Planned
**Created:** 2026-07-07
**Prior milestone:** v5.35 Real Pilot Blocker Burn-Down And Launch Execution

## Goal

Operate the first approved cohort, collect real parent/student/teacher/support evidence, and ship the highest-impact fixes needed before any expansion.

## Function Purpose

- Replace readiness assumptions with real activation, learning, support, billing, mobile, and AI quality signals.
- Keep the cohort small enough that product fixes and support intervention can happen quickly.
- Convert pilot incidents and friction into prioritized fixes with regression evidence.

## Implementation Strategy

- Start only if v5.35 returns `start_limited_pilot`.
- Run daily pilot review around activation, first learning action, support load, teacher SLA, billing/entitlement, notification delivery, AI quality, and mobile stability.
- Classify every finding as hotfix, product backlog, support intervention, documentation/copy change, or accepted limitation.
- Close with expand, hold, rollback, or remediate decision.

## Phases

- [ ] **Phase 352: First Cohort Live Operations** - Enable the approved cohort and track onboarding, login, entitlement, first learning action, and support intake.
- [ ] **Phase 353: Daily Feedback Incident And Metrics Review** - Run daily review across product, support, teacher, mobile, billing, AI, and notification evidence.
- [ ] **Phase 354: High-Impact Pilot Fix Implementation** - Implement the top pilot fixes with focused tests and support-visible release notes.
- [ ] **Phase 355: Pilot Learning Outcome And Support Quality Review** - Assess learning progress, teacher response quality, parent satisfaction, and unresolved friction.
- [ ] **Phase 356: Pilot Outcome Decision Gate** - Decide whether to expand, hold, roll back, or continue remediation.
