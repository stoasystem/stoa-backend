# Requirements: v5.7 Usage Ledger And Quota Reconciliation

**Milestone:** v5.7
**Status:** Planned
**Created:** 2026-07-03

## Requirements

### USAGE-01 Usage Ledger Contract

Acceptance criteria:

- Durable usage event schema includes actor user, subject/student, parent, product area, action, quantity, entitlement source, period key, idempotency key, timestamp, and display-safe metadata.
- Access patterns cover admin support lookup, parent/customer summaries, reconciliation, and future analytics handoff.
- Partition/sort key strategy is documented against current DynamoDB conventions.
- Retention/privacy boundaries are documented for support-visible metadata.
- Counter compatibility and reconciliation behavior are defined before writes are added.

### USAGE-02 Usage Event Repository And Service

Acceptance criteria:

- Repository writes usage events idempotently.
- Service exposes internal record/query/summary methods.
- Bounded summary queries support student, parent, period, product area, and entitlement source filters.
- Duplicate write attempts cannot double-count the same event.
- Tests cover repository write, duplicate write, query, and summary paths.

### USAGE-03 Plan-Governed Usage Instrumentation

Acceptance criteria:

- Question submission writes usage events with v5.6 entitlement source.
- OCR/AI answer generation writes distinguishable usage events.
- Teacher-help requests write usage events without double-counting retries.
- Chat and hint usage write events through the shared service.
- Instrumentation error behavior is explicit and test-covered where practical.

### USAGE-04 Quota Reconciliation And Admin Query

Acceptance criteria:

- Existing quota counters remain the enforcement path unless explicitly changed.
- Reconciliation can compare counter totals with ledger totals for a bounded period.
- Admin usage query filters by parent, student/user, period, product area, and entitlement source.
- Response shape is support-safe and excludes raw internal metadata by default.
- Tests cover ledger/counter consistency and mismatch reporting.

### VERIFY-40 v5.7 Usage Ledger Release Gate

Acceptance criteria:

- Usage ledger contract, repository/service, instrumentation, reconciliation, and admin query are complete.
- Focused backend tests pass.
- Release evidence records commit SHA, test command/output summary, and deferred product areas.
- Docs, roadmap, state, and remaining-feature queue are updated.
- v5.8 account verification handoff is updated.
