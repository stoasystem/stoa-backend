# v4.7 Requirements: Payment Production Activation And Provider Automation

**Status:** Active planning
**Created:** 2026-06-12
**Source:** `.planning/REQUIREMENTS.md`

## Goal

Activate production payment operations from the v4.4 readiness foundation through live provider checks, webhook registration readiness, direct refund execution, finance handoff, and explicit rollout controls.

## Requirements

### PAYACT-01 Payment Production Activation Contract And Provider Readiness

- Define live Stripe credential ownership, injection path, runtime validation, and redacted readiness evidence.
- Define live Standard/Premium price IDs, CHF expectations, TWINT eligibility requirements, and capability checks.
- Define webhook endpoint registration, webhook secret ownership, and required launch event set.
- Define finance acceptance expectations for invoices, refunds, tax/accounting handoff, dunning, and reconciliation.
- Define rollout gates for live checkout, direct refunds, and provider-readiness automation.

### PAYACT-02 Live Provider Readiness API Checks

- Add admin-only provider readiness checks for credential mode, price mapping, webhook readiness, TWINT eligibility, refund capability, and accounting metadata.
- Return fail-closed blocker states when credentials are absent or provider APIs fail.
- Redact secrets, raw provider payloads, payment details, and customer-sensitive payment data.
- Cover missing/test/live/provider-failure/success fixtures with focused tests.

### PAYACT-03 Direct Refund Execution And Finance Handoff

- Add controlled refund execution with eligible billing state, provider reference, operator reason, idempotency key, and admin authorization.
- Persist refund request, result, provider reference, lifecycle status, billing projection, and audit evidence.
- Expose refund status in parent/admin billing views without sensitive provider payloads.
- Add finance handoff export for invoice, refund, tax/accounting, payment method, reconciliation, and dunning metadata.

### PAYACT-04 Production Webhook Registration And Rollout Controls

- Verify endpoint mode, webhook secret availability, required event subscriptions, and last observed event status.
- Keep live checkout blocked until explicit rollout approval and provider readiness pass.
- Expose independent rollout controls for live checkout and direct refunds.
- Capture live-readiness status, blockers, and approved live smoke state.

### VERIFY-30 v4.7 Payment Activation Release Gate

- Run focused backend verification and relevant static checks.
- Verify provider readiness, refund execution, finance handoff, webhook readiness, and rollout controls.
- Update planning docs, feature gap audit, and remaining feature queue.
- Record live activation state: activated, blocked, deferred, or approved canary-only.
