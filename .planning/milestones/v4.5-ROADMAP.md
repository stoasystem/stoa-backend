# v4.5 Roadmap: Support Evidence Integrations And Operations Handoff

**Status:** Active planning
**Created:** 2026-06-12

## Goal

Turn the existing support-safe evidence package workflow into an approved destination handoff workflow with operator-visible status and refusal behavior for unapproved integrations.

## Phases

- [ ] **Phase 148: Support Destination Contract And Credential Readiness**
  - Define approved destination, credential, redaction, payload, attachment, readiness, and refusal contracts.
  - Requirement: SUPPORTINT-01.

- [ ] **Phase 149: Support Evidence Export Destination Integration**
  - Implement credential-gated destination delivery for support-safe evidence packages while keeping preview/copy/download fallback.
  - Requirement: SUPPORTINT-02.

- [ ] **Phase 150: Operator Queue And Handoff Status Visibility**
  - Add handoff lifecycle status, filters, detail visibility, retry behavior, and refusal evidence for operators.
  - Requirement: SUPPORTINT-03.

- [ ] **Phase 151: v4.5 Support Integration Release Gate**
  - Verify support integration behavior, refusal paths, redacted evidence, queue/status visibility, and remaining-feature updates.
  - Requirement: VERIFY-28.

## Execution Bias

Prioritize a narrow, approved integration path and usable operator workflow. Keep metadata-only boundaries intact, preserve manual fallback, and avoid broad CRM automation until the first destination workflow is proven.
