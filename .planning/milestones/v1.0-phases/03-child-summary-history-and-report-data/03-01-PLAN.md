## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Parent JWT to `/parents/me/children/{child_id}/summary` | Authenticated but untrusted caller supplies `child_id`; route must derive parent identity from JWT and local profile, not request path parent ids. |
| Parent route to DynamoDB repositories | Child-specific data leaves DynamoDB only after ownership verification. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01-S | Spoofing | `_resolve_parent_profile` and summary route dependency | mitigate | Use `Depends(require_role("parent"))` and local profile resolution before ownership checks. |
| T-03-01-I | Information Disclosure | `_get_owned_child_profile` and summary route | mitigate | Require child profile membership in `_scan_children_for_parent(resolved.parent_user_id)` before any data reader call. |
| T-03-01-T | Tampering | `child_id` path parameter | mitigate | Treat `child_id` as selector only; never use it to derive parent ownership. |
| T-03-01-D | Denial of Service | Summary aggregation | accept | Existing context accepts bounded MVP reads; use fixed limits for questions and report candidates. |
| T-03-01-SC | Tampering | npm/pip/cargo installs | accept | No package-manager installs are part of this plan. |
</threat_model>
