# Requirements: v5.18 Warehouse BI Observability And Product Analytics Activation

**Milestone:** v5.18
**Status:** Planned
**Created:** 2026-07-05

## Requirements

### BI-01 Analytics Reality Audit And Taxonomy

- Map existing warehouse-readiness exports, usage reconciliation, billing readiness, curriculum analytics, notification delivery, support handoff, and core smoke events.
- Define dimensions for live, blocked, read-only, safe-fixture, local-only, and failed states.
- Confirm privacy boundaries before any export activation.

### BI-02 Warehouse Export Job Activation

- Implement or enable repeatable aggregate export jobs.
- Exports are idempotent, bounded, and support-safe.
- Backfill, retry, and partial-failure behavior is documented.

### BI-03 Operator Dashboards

- Dashboards cover usage/quota, billing/provider readiness, curriculum migration/editor health, teacher-help/support load, notification delivery, and release smoke outcomes.
- Dashboards show blocker categories and support actions.
- No dashboard exposes raw student content or provider payloads.

### BI-04 APM And Alert Routing

- Add low-cardinality error/status dimensions for core flows.
- Route alerts to internal operations without external customer messaging unless separately approved.
- Alerts classify provider blockers separately from product regressions.

### VERIFY-52 BI Observability Release Gate

- Focused backend/frontend checks pass for export/dashboard contracts.
- Runbooks and release evidence are updated.
- Remaining BI/provider/APM limitations are explicit.
