# Requirements: v3.2 Content Moderation And Internal Operations

**Milestone:** v3.2
**Status:** Active
**Created:** 2026-06-08

## Goal

Close the remaining MVP admin content moderation workflow from `stoa_docs` with report actions, moderation cases, admin queue/detail/actions, and lightweight functional verification.

## Requirements

| Requirement | Phase | Status |
|-------------|-------|--------|
| MOD-01 Content moderation contract and data model readiness | Phase 96 | Planned |
| MOD-02 Backend moderation reporting and admin APIs | Phase 97 | Planned |
| UI-17 Moderation reporting and admin queue UI | Phase 98 | Planned |

## Product Build Priority

This milestone is an internal development milestone. Build the workflow first: users can report, admins can triage, and cases can be resolved. Verification should prove the feature works, not produce broad compliance evidence.

## Out Of Scope

- Stripe/TWINT payment integration.
- Multi-subject curriculum expansion.
- Student memory/personalization.
- AI teacher assistance tools.
- WebSocket realtime notifications.
- Compliance-grade moderation/legal operations.
