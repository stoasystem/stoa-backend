# Requirements: v5.15 Usage, Quota, And Product Stability

**Milestone:** v5.15
**Status:** Planned
**Created:** 2026-07-05

## Requirements

### STABILITY-01 Usage Reality Audit

- Map all current usage-bearing flows and identify whether they emit ledger events, counters, both, or neither.
- Mark each flow as implemented, missing, intentionally skipped, or blocked by absent frontend/backend route.
- Define priority fixes for paid access and student-facing quota behavior.

### STABILITY-02 Ledger Coverage And Idempotency

- Major successful usage flows emit privacy-safe, idempotent events.
- Failed, preview, admin-triggered, retry, and duplicate flows follow documented skip/idempotency rules.
- Tests cover duplicate request IDs and partial failure behavior.

### STABILITY-03 Quota Reconciliation And Explanations

- Ledger rows, aggregate counters, entitlement limits, and user/admin summaries can be reconciled.
- Drift is detected and surfaced in support-safe form.
- Parent/admin explanations do not expose raw learning content or provider payloads.

### STABILITY-04 Core Health And Smoke Gates

- Smoke scripts or health checks cover login, entitlement, curriculum read, question submit, teacher help, and admin support surfaces.
- Checks return actionable request IDs or metadata.
- Failure modes are documented for internal development handoff.

### VERIFY-49 Usage And Stability Gate

- Focused tests pass for usage coverage, reconciliation, smoke checks, and support summaries.
- Docs and release evidence record remaining BI/APM/provider dependencies.
