# v4.2 Requirements: Production Notification Delivery Readiness

**Status:** Active planning
**Started:** 2026-06-11
**Source:** `.planning/REQUIREMENTS.md`

## Goal

Promote STOA's local realtime notification foundation into production-deliverable notification capability through backend/infrastructure contracts, delivery operations, durable user preferences, email digest readiness, and focused release evidence.

## Requirements

### NOTIFYDEL-01 Production Notification Infrastructure Contract

- Define production WebSocket endpoint shape, API Gateway route/integration expectations, environment variables, deployment ownership, and fallback behavior.
- Map notification events to in-app realtime, polling fallback, digest readiness, and push-ready preference channels.
- Define delivery state fields for internal rollout and operator debugging.
- Separate backend, CDK, frontend, and native ownership boundaries.

### NOTIFYDEL-02 WebSocket Delivery Operations And Preference APIs

- Add durable notification category/channel preference reads and updates.
- Preserve existing in-product notification behavior by default.
- Add delivery decision/status behavior for realtime, in-app fallback, digest-ready, and push-ready handling.
- Expose bounded operator-visible delivery health/status signals.

### NOTIFYDEL-03 Email Digest And Push Preference Readiness

- Add digest-ready selection or preview behavior by recipient, category, and time window.
- Define stable metadata-safe digest payloads for future email templates.
- Store and surface push/native preference flags while provider delivery remains optional.
- Prove no-provider fallback behavior with tests or fixtures.

### VERIFY-25 v4.2 Functional Release Gate And Notification Delivery Audit

- Run focused backend verification and relevant static checks.
- Update planning, feature-gap, and remaining-feature docs.
- Capture available build/deploy/CDK/API/browser evidence or record production verification deferral.
- Recommend the next milestone from the remaining feature queue.
