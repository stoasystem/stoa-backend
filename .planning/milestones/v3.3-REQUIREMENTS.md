# Requirements: v3.3 Subscription Operations MVP

**Milestone:** v3.3
**Status:** Complete
**Created:** 2026-06-08

## Goal

Make the MVP manual subscription model usable before Stripe/TWINT integration.

## Requirements

| Requirement | Phase | Status |
|-------------|-------|--------|
| SUBOPS-01 Subscription operations contract and entitlement model | Phase 100 | Complete |
| SUBOPS-02 Backend subscription request and admin tier APIs | Phase 101 | Complete |
| UI-18 Parent subscription management and admin queue | Phase 102 | Complete |
| VERIFY-16 v3.3 functional release gate and billing readiness | Phase 103 | Complete |

## Product Build Priority

This milestone turns the current manual paid onboarding process into an internal product workflow: parents can express subscription intent, and admins can process requests and apply tiers. It intentionally stops before payment-provider integration.

## Out Of Scope

- Stripe/TWINT integration.
- Payment webhooks, invoices, refunds, taxes, or automated payment reconciliation.
- Broad multi-subject rollout.
- Student memory/personalization.
- Realtime/WebSocket notifications.
