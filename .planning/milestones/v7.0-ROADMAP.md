# Roadmap: v7.0 Final Live Approval And Controlled Expansion Start

**Status:** Planned
**Created:** 2026-07-09
**Prior milestone:** v6.9 Public Launch Decision And Market Readiness

## Goal

Turn v6.9 market-readiness contracts into a real controlled-expansion start or an explicit hold. v7.0 must gather final owner approval, current production evidence, provider/mobile/support readiness, cohort scope, and rollback controls before enabling a larger controlled cohort.

## Reality Check

v6.9 is complete as a market-readiness decision contract. It does not approve public launch, paid marketing, uncontrolled provider writes, or broad expansion by itself. Public launch prep is possible only with healthy evidence and final owner approval; paid marketing remains separately gated.

The next work should therefore start v7, but v7.0 must be controlled expansion execution, not a public-launch claim.

## Function Purpose

- Convert v6.9 launch/expansion decision inputs into final real approval.
- Start a larger controlled cohort only if current evidence is healthy.
- Keep broad public launch and paid marketing blocked unless later milestones explicitly approve them.
- Preserve rollback, pause, support staffing, provider disablement, and user-visible limitation controls.

## Implementation Strategy

- Refresh final owner approval, scope, support capacity, provider/mobile state, revenue reconciliation, learning quality, and incident readiness.
- Run production checks read-only by default; allow only explicitly approved, scoped, reversible pilot/expansion actions.
- Record redacted evidence with timestamps, owners, request/build IDs, rollout flags, cohort aliases, blocker states, and rollback authority.
- Close with controlled expansion start, hold, rollback, or another remediation cycle.
