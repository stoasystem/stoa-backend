# Plan 21-01: Explicit Smoke and Partial Artifact Cleanup

**Phase:** 21 - Smoke and Orphan Artifact Cleanup
**Status:** Planned
**Requirements:** CLEAN-01, CLEAN-02, CLEAN-03

## Objective

Add explicit cleanup paths for deterministic smoke artifacts and failed partial report artifact writes, then deploy and prove live smoke no longer leaves the smoke object behind.

## Tasks

1. Update report artifact helper cleanup behavior.
   - After JSON write succeeds and HTML write fails, call `delete_object` for the JSON key before re-raising.
   - Do not delete either artifact after successful real report writes.
   - In smoke, delete the deterministic JSON object after successful readback.
   - Return `cleanup: performed` or `cleanup: failed` in smoke output.

2. Update tests.
   - Extend the fake S3 client with delete tracking and optional failure injection.
   - Assert successful real report writes do not delete objects.
   - Assert HTML write failure deletes the JSON partial object and re-raises.
   - Assert smoke writes, reads, deletes, and reports cleanup status.

3. Deploy updated Lambda code.
   - Rebuild `dist/stoa` from `src/stoa`.
   - Update `stoa-api` and `stoa-weekly-report` Lambda code or deploy through CDK with the refreshed asset.
   - Wait for both Lambda updates to complete.

4. Live verification.
   - Invoke `stoa-weekly-report` smoke.
   - Confirm smoke output reports cleanup performed.
   - Confirm `head-object` for the deterministic smoke key returns not found after smoke.

5. Write execution artifacts and update planning state.
   - Create `21-01-SUMMARY.md`, `21-VERIFICATION.md`, and `21-REVIEW.md`.
   - Mark CLEAN-01 through CLEAN-03 complete.
   - Advance state to Phase 22.
