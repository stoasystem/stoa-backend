# Plan 22-01: Admin Report Operations Visibility and Resend

**Phase:** 22 - Report Operations Visibility and Recovery
**Status:** Planned
**Requirements:** OPS-01, OPS-02, OPS-03, OPS-04

## Objective

Add admin-only backend operations to inspect report artifact/delivery metadata and resend a specific failed report email without exposing raw private artifacts or regenerating unrelated reports.

## Tasks

1. Add internal private HTML artifact read support.
   - Add `get_report_html` to `report_artifact_service`.
   - Validate canonical `weekly-reports/.../report.html` keys.
   - Decode bytes or string body safely.

2. Add admin report operations endpoints.
   - `GET /admin/reports/{parent_id}/{student_id}/{week_start}/ops` returns metadata, artifact keys, status, delivery fields, and operation audit fields.
   - `POST /admin/reports/{parent_id}/{student_id}/{week_start}/resend` resends only failed delivery reports.
   - Return no raw report content, public URLs, or presigned URLs.

3. Add audit/status behavior.
   - On resend success, update `status=email_sent`, `email_status=sent`, `email_sent_at`, `updated_at`, `last_operation=resend_email`, `last_operation_result=success`, `last_operation_by`.
   - On resend failure, keep/mark `email_failed`, record error class/message, and audit failure.
   - Include operator identity from admin claims where available.

4. Add tests.
   - Metadata endpoint is admin-only and does not expose raw/private content URLs.
   - Resend targets one failed report and uses the existing HTML artifact.
   - Resend refuses successful/non-failed reports.
   - Audit fields are persisted.

5. Deploy and verify.
   - Run focused backend tests.
   - Rebuild Lambda package, deploy code, sync CDK asset, and run `cdk diff` clean.

## Acceptance Criteria

- OPS-01: Admin can inspect report artifact metadata and delivery status for a parent/student/week without S3 console use.
- OPS-02: Admin can resend a specific failed report delivery without regenerating unrelated successful reports.
- OPS-03: Operations are admin-only and responses expose no public S3 URLs or raw private report content.
- OPS-04: Resend attempts persist enough audit/status fields for support triage.
