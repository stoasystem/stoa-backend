# Plan 171-01: Define Native Mobile And Localization Governance Contract

## Goal

Create a concrete v5.0 contract for native/mobile rollout readiness, full localization governance, translation QA, native notification/offline handoff, and client release evidence.

## Work Items

1. Inspect v4.1 backend mobile/locale foundations, v4.3 frontend mobile/localization evidence, and v4.9 notification/native handoff.
2. Define backend, frontend, native, localization, content, and release ownership boundaries.
3. Define mobile-critical user flows for student, parent, tutor, and admin roles.
4. Define supported locale policy, fallback behavior, translation source of truth, key naming, copy ownership, and review workflow.
5. Define native push token, offline/read-through, deep-link, reconnect, and permission-state handoff expectations.
6. Define mobile API readiness targets for auth/session, profile/locale, learning, reports, tutor tools, notifications, billing, and support.
7. Define release evidence and rollout states for contract-ready, frontend-ready, native-ready, blocked, and deferred.
8. Write Phase 172 through 175 implementation handoff.

## Verification

- Contract maps every MOBILELOC-01 acceptance criterion to implementation targets.
- Cross-workspace ownership is explicit and actionable.
- Full native app binary implementation is not required for Phase 171.
- Follow-up phases have concrete code, docs, and test targets.
