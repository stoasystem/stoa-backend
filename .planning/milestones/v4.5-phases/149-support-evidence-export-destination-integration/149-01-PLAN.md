<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| admin request -> delivery route | Destination mode and operator reason must not bypass admin auth, readiness gates, fail-fast unknown checks, or refused-record handling for contract-defined unapproved modes. |
| support package -> delivery record | Metadata-safe package output must be reduced to allowlisted summaries and references before persistence. |
| delivery service -> DynamoDB | Stored lifecycle rows must be metadata-only, idempotent, and separated from package audit semantics. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-149-01 | Elevation of Privilege | delivery endpoint | mitigate | Keep route behind `require_role(\"admin\")`; test non-admin requests do not read evidence. |
| T-149-02 | Information Disclosure | delivery payload/records | mitigate | Persist only allowlisted metadata, run privacy marker checks, and test absence of artifact keys, presigned URLs, headers, cookies, tokens, and credentials. |
| T-149-03 | Repudiation | delivery lifecycle | mitigate | Persist delivery summary and append-only audit rows with actor, correlation ID, idempotency key, retry count, status, and redacted reasons. |
| T-149-04 | Denial of Service | duplicate delivery clicks/retries | mitigate | Use stable idempotency independent of UUID package ID and reuse existing delivery records. |
| T-149-05 | Tampering | destination mode | mitigate | Allow only `internal_queue` to reach queued state; keep contract-defined third-party/generic external modes as refused delivery records before evidence reads and unknown modes as fail-fast 422. |
| T-149-SC | Tampering | package installs/provider SDKs | accept | No new dependency or third-party provider SDK is required for Phase 149. |
</threat_model>
