# Plan 67-01: Backend Support Handoff Package APIs

## Objective

Implement admin-only backend support handoff package generation with metadata-only evidence composition, unsupported external destination refusal, redacted audit metadata, and focused tests.

## Tasks

1. Add a support handoff service that composes package sections from existing sanitized recovery support packages, release evidence validation, fixture status, and operator notes.
2. Add append-only support handoff audit persistence using the existing DynamoDB table and conditional write pattern.
3. Add an admin-only POST endpoint under `/admin/reports/support-handoff-package`.
4. Enforce destination policy: allow `preview`, `copy`, and `download`; refuse `external_write`; reject unknown destination modes before evidence reads.
5. Validate missing evidence references as structured package validation metadata and keep outputs allowlisted.
6. Add focused tests for admin-only auth, successful package generation, audit metadata, redaction denylist, missing references, external write refusal, and unknown destination rejection.
7. Run focused backend tests and update GSD phase verification/summary artifacts.
