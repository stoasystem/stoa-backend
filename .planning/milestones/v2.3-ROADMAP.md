## Current Milestone

### v2.3 Release Evidence Automation And Fixture Lifecycle

**Milestone Goal:** Operators can produce repeatable, redacted release evidence bundles and manage the named non-customer safe fixture lifecycle without expanding production mutation scope.

This milestone turns the manually assembled v2.1/v2.2 release evidence into a documented, repeatable operator workflow. It keeps production smoke read-only by default, limits mutation verification to approved safe fixtures, and avoids new infrastructure unless Phase 62 proves current repo scripts, API endpoints, and CDK resources cannot support the evidence workflow.

## Phase Details

### Phase 62: Release Evidence Contract And Safe Fixture Lifecycle Plan

**Goal**: Implementers have a precise release evidence contract, redaction/privacy model, safe-fixture lifecycle plan, and infrastructure decision before automation code.
**Depends on**: Phase 61
**Requirements**: EVIDENCE-AUTO-01, FIXTURE-02
**Success Criteria**:

1. Evidence contract defines required release bundle fields for backend deploy, frontend deploy, Lambda manifest/runtime, CDK diff/deploy, admin-only API checks, browser smoke, request IDs, commit SHAs, timestamps, and privacy denylist results.
2. Redaction model forbids auth tokens, passwords, S3 keys, presigned URLs, raw report JSON/HTML, private artifact payloads, and customer-identifying fixture data from committed evidence.
3. Safe-fixture lifecycle plan defines fixture identity, ownership, status inspection, allowed mutation modes, cleanup/restore evidence, retention policy, and refusal behavior.
4. CDK readiness classifies whether current resources are sufficient or exactly what infrastructure change is required.

**Plans**: 1 complete
**Completed**: 2026-06-06

### Phase 63: Backend Release Evidence And Fixture Inventory Tooling

**Goal**: Operators can collect release evidence and inspect fixture state through repeatable backend tooling.
**Depends on**: Phase 62
**Requirements**: EVIDENCE-AUTO-02, FIXTURE-02
**Success Criteria**:

1. Tooling collects or validates deploy run IDs, commit SHAs, Lambda runtime state, CDK diff classification, API request IDs, and smoke summaries into the Phase 62 evidence schema.
2. Tooling inspects the named safe fixture and reports sanitized artifact/version/rollback status without private S3 keys or raw artifact payloads.
3. Tooling refuses production mutation unless an explicit approved fixture name and mutation mode are supplied.
4. Tests cover schema validation, redaction, refusal behavior, fixture inventory output, and missing-evidence failures.

**Plans**: 1 complete
**Completed**: 2026-06-06

### Phase 64: Admin Release Evidence And Fixture Status UI

**Goal**: Admins can view sanitized release evidence and safe-fixture status from `/admin/report-operations`.
**Depends on**: Phase 63
**Requirements**: UI-10
**Success Criteria**:

1. UI exposes admin-only release evidence and fixture status controls without triggering mutation.
2. UI renders sanitized bundle metadata, fixture status, request IDs, commit SHAs, and validation failures.
3. UI never renders private S3 keys, presigned URLs, raw report JSON/HTML, auth tokens, or raw artifact payloads.
4. Playwright covers evidence/status rendering, error states, admin-only gating, and privacy denylist.

**Plans**: 1 complete
**Completed**: 2026-06-06
**UI hint**: yes

### Phase 65: v2.3 Release Gate And Milestone Audit

**Goal**: v2.3 closes with evidence proving release evidence automation is deployed, admin-only where applicable, privacy-safe, and production-verified without customer-impacting mutation.
**Depends on**: Phase 64
**Requirements**: VERIFY-06
**Success Criteria**:

1. Release gate records backend/frontend deploy runs, commit SHAs, Lambda manifest/runtime state, CDK diff/deploy evidence, local quality gates, API request IDs, and browser smoke results.
2. Production smoke is read-only by default and verifies route/auth/privacy/bundle markers without customer artifact mutation.
3. Any fixture mutation verification uses only the approved named non-customer fixture and records cleanup/restore evidence.
4. Final v2.3 audit records residual risks, rollback path, and future requirements.

**Plans**: 1 complete
**Completed**: 2026-06-06
