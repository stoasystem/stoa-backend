# Phase 89 Plan

## Deliverables

1. Add Cognito-backed forgot-password and reset-password endpoints.
2. Make email verification behavior explicit in registration metadata and auth responses.
3. Add formal parent-student binding records.
4. Use formal bindings in parent portal reads and weekly report discovery, with legacy `student.parent_id` fallback.
5. Add admin-only binding inspection/repair endpoints.
6. Cover auth reset edge cases, parent-child authorization, admin repair, and token non-leakage with tests.

## Security Decisions

- Forgot-password for unknown local profiles returns `accepted` without calling Cognito.
- Reset-password normalizes code/user/password Cognito errors to a generic invalid reset response.
- Registration activates a parent-student link only when both profiles cross-reference each other by email.
- One-sided registration claims are stored as pending binding statuses and require admin repair or the other profile to confirm.
- Admin repair mirrors the formal binding into the legacy `student.parent_id` field for compatibility.
