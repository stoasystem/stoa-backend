# Phase 90 Plan

## Deliverables

1. Add edit-before-AI OCR correction support through optional `corrected_text`.
2. Preserve final content, original submitted content, corrected text, OCR text, and OCR metadata in the stored question row.
3. Sanitize question API responses so private image keys are not returned.
4. Replace bounded question-history quota checks with an atomic daily usage counter.
5. Add tests for corrected OCR flow, image/text submission, quota boundaries, and authorization/privacy.

## Decisions

- The correction flow is edit-before-AI: if `corrected_text` is provided, it becomes the final AI input.
- If no correction is provided, OCR text continues to be appended to the submitted content.
- OCR metadata returned to clients includes status, source type, text length, correction flag, and failure class only.
- Raw `image_s3_key` remains stored for backend audit/debug but is forced to `null` in API responses.
