# v4.2 Roadmap: Production Notification Delivery Readiness

**Status:** Active planning
**Started:** 2026-06-11

## Goal

Move STOA notification capability from local realtime foundations toward production-deliverable internal rollout: production WebSocket delivery contract, preference APIs, delivery operations, digest readiness, and release evidence.

## Phases

- [ ] **Phase 136: Production Notification Infrastructure Contract**
  - Define production endpoint, API Gateway route/integration, environment, channel mapping, delivery state, and ownership boundaries.
  - Requirement: NOTIFYDEL-01.

- [ ] **Phase 137: WebSocket Delivery Operations And Preference APIs**
  - Implement durable notification preference APIs and delivery decision/status behavior.
  - Requirement: NOTIFYDEL-02.

- [ ] **Phase 138: Email Digest And Push Preference Readiness**
  - Implement digest selection/preview readiness and push preference metadata without requiring live provider credentials.
  - Requirement: NOTIFYDEL-03.

- [ ] **Phase 139: v4.2 Functional Release Gate And Notification Delivery Audit**
  - Verify focused behavior, update docs, capture evidence, and recommend the next milestone.
  - Requirement: VERIFY-25.

## Execution Bias

This is an internal development milestone. Prioritize usable feature construction and practical verification. Do not expand broad security/compliance test scope unless required by the specific feature surface or production traffic boundary.
