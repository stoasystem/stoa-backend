# Plan 55-01: Backend Artifact Edit Preview And Versioned Apply APIs

## Objective

Implement admin-only artifact edit preview/apply APIs with versioned S3 artifact writes, stale-source rejection, sanitized responses, and append-only audit evidence.

## Tasks

1. Add versioned report artifact key/delete helpers under the canonical private `weekly-reports/` prefix.
2. Add report repository helpers for artifact edit draft rows and conditional artifact metadata updates.
3. Implement a new artifact edit service that validates allowlisted fields, reads current JSON artifacts, builds sanitized diffs, writes versioned JSON/HTML artifacts, updates report summary pointers, and writes redacted audit evidence.
4. Add admin route request/response models and preview/read/apply endpoints.
5. Add focused tests for admin-only access, private marker rejection, sanitized preview, versioned apply, stale-source rejection, rollback metadata, and audit evidence.
