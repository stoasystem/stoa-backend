# Roadmap: v2.8 CDK-Managed Immutable Evidence Storage Deployment

**Status:** Active
**Created:** 2026-06-07

## Goal

Deploy and enable CDK-managed immutable evidence storage for report operations retention manifests while preserving the metadata-only privacy boundary and fail-closed safety behavior shipped in v2.7.

## Release Gate

v2.8 is not complete until release evidence records:

- Lambda build manifest and backend runtime state.
- Backend deploy run ID, commit SHA, and timestamp.
- Infra deploy run ID, commit SHA, and timestamp.
- CDK diff/deploy evidence for immutable storage resource, Lambda env vars, and IAM scope.
- Admin-only API request IDs and redacted payload samples.
- Production browser smoke for `/admin/report-operations`.
- Privacy denylist results proving no report artifact, S3 key, presigned URL, raw JSON/HTML, auth token, cookie, password, or AWS secret exposure.
- Final milestone audit with residual compliance/legal gaps.

## Safety Boundaries

- No manual AWS console changes.
- No production audit deletion.
- No customer report artifact mutation.
- No direct external support-system write.
- No broad compliance claim beyond verified CDK-managed immutable evidence storage behavior.
