## Release Gate

v2.8 is not complete until release evidence records:

- Lambda build manifest and backend runtime state.
- Backend deploy run ID, commit SHA, and timestamp.
- Infra deploy run ID, commit SHA, and timestamp.
- CDK diff/deploy evidence for immutable storage resource, Lambda env vars, and IAM scope.
- Admin-only API request IDs and redacted payload samples.
- Production browser smoke for `/admin/report-operations`.
- Privacy denylist results proving no report artifact, S3 key, presigned URL, raw JSON/HTML, auth token, cookie, password, or AWS secret exposure.
- Final milestone audit with residual compliance/legal gaps.

Status: complete. Evidence is archived in `.planning/milestones/v2.8-phases/82-v2.8-release-gate-and-live-immutable-storage-verification/`.

## Safety Boundaries

- No manual AWS console changes.
- No production audit deletion.
- No customer report artifact mutation.
- No direct external support-system write.
- No broad compliance claim beyond verified CDK-managed immutable evidence storage behavior.

## Key Outcomes

- Deployed a retained, versioned, Object Lock-enabled immutable evidence metadata bucket through CDK.
- Injected CDK-managed immutable storage settings into `stoa-api`.
- Scoped API IAM to `s3:GetObject` and `s3:PutObject` on the approved metadata prefix.
- Verified live S3 Object Lock default retention, encryption, logging, public access block, Lambda env vars, and IAM.
- Verified backend env-driven readiness, duplicate/reference-exists refusal, create-only object writer behavior, privacy checks, and failure handling.
- Persisted one approved metadata-only immutable manifest in production and verified API, DynamoDB, S3 Object Lock headers, and browser smoke.

## Residual Gaps

- The 365-day GOVERNANCE retention period still needs formal compliance/legal approval.
- Operational legal-hold procedure still needs owner assignment, runbook, review cadence, and emergency break-glass policy.
