# Plan 25-01: Bulk Email Resend Recovery

## Objective

Add an admin-only bulk resend endpoint that retries selected `email_failed` reports independently and returns per-report results without exposing private report artifacts.

## Tasks

1. Extend admin request/response models
   - Add selected report identifier model.
   - Add bulk request model with min/max item constraints.
   - Add per-item result and aggregate response models.

2. Extract resend helper
   - Keep single-report resend behavior unchanged.
   - Move private artifact read, email send, and audit update into a reusable helper.
   - Keep failure audit/error redaction intact.

3. Add bulk resend endpoint
   - Add `POST /admin/reports/bulk-resend`.
   - Require admin role.
   - Process selected items in order.
   - Return `success`, `refused`, `not_found`, or `failed` per item.
   - Do not let one failed/refused item block remaining items.

4. Add tests
   - Non-admin requests are rejected.
   - Batch size is enforced.
   - Mixed success/refused/not-found/failed batch returns per-item results and continues.
   - Responses do not include raw HTML, private S3 keys, or direct URLs.

5. Verification
   - Run focused pytest and ruff.
   - Write summary, review, and verification artifacts.
   - Update roadmap, requirements, and state for Phase 25 completion.

## Acceptance Criteria

- DEL-01 through DEL-04 are covered.
- Bulk resend is admin-only.
- Selected items are independently validated and reported.
- Backend reads private HTML artifacts; clients receive only operation metadata.
- Per-report audit fields are written for success and failure.
