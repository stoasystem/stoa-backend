# v4.7 Roadmap: Payment Production Activation And Provider Automation

**Status:** Active planning
**Created:** 2026-06-12
**Source:** `.planning/ROADMAP.md`

## Goal

Turn payment readiness into controlled production activation with live provider checks, webhook registration readiness, direct refunds, finance handoff, and rollout controls.

## Phases

- [ ] **Phase 156: Payment Production Activation Contract And Provider Readiness**
  - Define credential ownership, price/TWINT readiness, webhook registration, finance acceptance, and rollout gates.
  - Requirement: PAYACT-01.

- [ ] **Phase 157: Live Provider Readiness API Checks**
  - Add admin-only readiness checks for credentials, prices, TWINT, webhook, refunds, and accounting metadata.
  - Requirement: PAYACT-02.

- [ ] **Phase 158: Direct Refund Execution And Finance Handoff**
  - Add controlled refund execution, idempotency, audit evidence, billing projection updates, and finance handoff export.
  - Requirement: PAYACT-03.

- [ ] **Phase 159: Production Webhook Registration And Rollout Controls**
  - Add webhook readiness verification and independent rollout controls for live checkout and refunds.
  - Requirement: PAYACT-04.

- [ ] **Phase 160: v4.7 Payment Activation Release Gate**
  - Verify activation readiness, docs, release evidence, and next milestone recommendation.
  - Requirement: VERIFY-30.

## Execution Bias

Prioritize production activation features. Real customer charging stays behind explicit approval, but the milestone should produce working provider automation and operator controls rather than only documentation.
