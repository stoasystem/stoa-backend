# Phase 212 Plan: Email Verification Contract And Account State Policy

## Objective

Define and implement the shared backend contract for email verification state without yet relying on broad operations-console work.

## Tasks

1. Add a verification policy helper that defines states, activation status, legacy compatibility, resend cooldown semantics, and public/support summaries.
2. Add public auth response fields for email verification and account activation state.
3. Define parent/student binding behavior for unverified parties.
4. Document the route-level policy and test matrix.
5. Verify with focused auth lifecycle tests and Ruff.

## Acceptance Mapping

| Success Criteria | Evidence |
|------------------|----------|
| States cover registered, unverified, pending, verified, expired, resend-limited, and blocked. | `account_verification_service.py`, `212-EMAIL-VERIFICATION-CONTRACT.md` |
| Registration/profile response shape avoids provider internals. | `UserOut`, `AuthResponse`, focused tests |
| Parent/student binding behavior is explicit. | Binding helper changes and focused tests |
| Route policy identifies allowed/blocked unverified behavior. | Contract and auth route tests |
| Test matrix is documented. | Contract test matrix and `tests/test_auth_account_lifecycle.py` |
