# Phase 213 Plan: Registration Verification Enforcement

## Objective

Make new registrations and login-compatible account lifecycle paths obey the verification policy.

## Tasks

1. Replace registration's backend-created `email_verified=true` behavior with Cognito `sign_up`.
2. Persist initial verification metadata and public activation status on the user profile.
3. Return no token from registration while verification is pending.
4. Block login token return for profiles that require verification.
5. Update parent/student binding creation to avoid silently granting fully active access.
6. Add focused tests for pending and verified states.
