# Plan 59-01: Backend Artifact Rollback APIs And Fixture Harness

## Objective

Implement admin-only artifact rollback preview/apply APIs and a named safe-fixture smoke harness while preserving artifact privacy and stale-source safety.

## Tasks

1. Add repository helpers for rollback preview rows.
2. Add rollback preview/read/apply service behavior with stale checks, pointer switching, sanitized responses, and audit events.
3. Add admin route request/response models and rollback endpoints.
4. Add `rollback_artifact` action eligibility metadata.
5. Add safe-fixture smoke harness with explicit refusal behavior.
6. Add focused tests for authorization, preview validation, apply metadata, stale rejection, audit, and privacy denylist.
