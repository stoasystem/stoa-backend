# Roadmap: v5.6 Native Mobile App And Offline Push Readiness

**Status:** Active planning
**Created:** 2026-06-16

Current active roadmap lives in `.planning/ROADMAP.md`.

## Goal

Turn completed mobile/API, notification, assignment automation, learning operations, and teacher-dispatch readiness into a native mobile app and offline/push implementation milestone.

## Purpose

The user-facing product gap is no longer whether STOA has backend concepts for mobile, notifications, assignments, and teacher dispatch. The gap is whether those capabilities can become a coherent native app experience for students, parents, teachers/tutors, and operators.

## Implementation Strategy

- Reuse v5.0 native/mobile contracts, v4.9 notification/native delivery readiness, v5.3 controlled assignment automation, v5.4 learning operations frontend contracts, and v5.5 dispatch metadata.
- Define or implement app shell, auth/session, role homes, API calls, push-token lifecycle, deep-link payloads, offline read-through cache boundaries, and release evidence.
- Keep internal development moving while treating live APNS/FCM credentials, app-store publication, final live payments, and external support provider activation as separate prerequisites.
