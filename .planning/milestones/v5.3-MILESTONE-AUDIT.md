The integration checker initially found one blocker: subject-scoped previews could not be executed because execute recomputed preview without the original `subject`.

Resolution:

- `AssignmentAutomationExecuteRequest` now accepts optional `subject`.
- Execute route passes `subject` into the service.
- Service recomputes server-side preview with the same subject scope before batch/candidate comparison.
- Regression test covers subject-scoped preview, execute, assignment creation, and parent-safe automation visibility.
