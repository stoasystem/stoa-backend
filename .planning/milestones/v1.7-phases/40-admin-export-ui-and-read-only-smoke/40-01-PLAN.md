# Plan 40-01: Admin Export UI And Read-only Smoke

**Phase:** 40 - Admin Export UI And Read-only Smoke
**Milestone:** v1.7 Recovery Evidence Export & Admin Credential Operations
**Status:** Complete
**Created:** 2026-06-05

## Objective

Add a read-only recovery evidence export panel to `/admin/report-operations`, verify it can call the Phase 39 backend contract, and prove the UI does not expose private report artifact markers.

## Tasks

1. Add frontend types and API function for `GET /admin/reports/recovery-evidence`.
2. Add a React Query mutation hook for evidence export.
3. Add a `RecoveryEvidencePanel` to `AdminReportOperationsPage`.
4. Support:
   - export selected recovery job
   - export recent jobs
   - JSON copy
   - JSON download
   - metadata counts and request ID display
5. Extend `tests/e2e/admin-report-operations.spec.ts` with export route mocks and assertions.
6. Verify with lint, build, focused e2e, and local browser smoke.

## Acceptance Criteria

- Admin UI can load metadata-only recovery evidence.
- Export controls are read-only and separate from mutation actions.
- E2E verifies selected-job and recent-jobs export.
- UI body text does not include `weekly-reports/`, S3 key field names, presigned URLs, or S3 URLs.
- Frontend build and lint pass.
- Local browser smoke confirms export panel and JSON preview render.
