# Plan 178-01: Define Production Content Migration Pipeline And Validation

## Goal

Define a repeatable production content migration pipeline contract with dry-run, apply, validation, conflict detection, evidence, and rollback behavior.

## Work Items

1. Define source manifest schema and batch metadata.
2. Define subject/topic/public ID/version ID mapping rules.
3. Define dry-run behavior, result categories, conflict detection, and validation output.
4. Define apply gating, version writes, migration evidence, audit events, and publish sequencing.
5. Define rollback/undo metadata and published-content safety rules.
6. Define test targets and frontend/operator handoff points.

## Verification

- Migration contract maps to CURRICULUMXP-03 acceptance criteria.
- Dry-run is explicitly non-mutating.
- Apply requires operator approval and writes evidence.
- Rollback protects existing published content.
