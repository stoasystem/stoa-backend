# Requirements: v8.1 Live Rollout Operations And Incident Remediation

**Milestone:** v8.1
**Status:** Completed
**Created:** 2026-07-09
**Prior milestone:** v8.0 External Rollout Approval And Live Evidence Execution

## Purpose

Operate the live rollout or remediate the v8.0 hold blockers. v8.1 must convert live evidence into fixes, support actions, rollback decisions, and growth-readiness signals.

## Requirements

### V8OPS-01 Live Rollout Day-One Operations

Acceptance criteria:

- Rollout dashboard, support room, teacher coverage, incident watch, revenue watch, learning watch, mobile/provider watch, and rollback readiness are active.
- Evidence distinguishes real users from internal, test, fixture, and dry-run traffic.
- Severe issues trigger pause, rollback, hotfix, provider disablement, or support escalation review.
- Daily operating cadence is owner-assigned.

### V8OPS-02 Live Account Revenue Support Incident Fixes

Acceptance criteria:

- Auth, verification, entitlement, usage/quota, billing, lifecycle, support, refund, and account clarity issues are fixed or explicitly deferred.
- Revenue-impacting fixes are auditable, reversible, and support-visible.
- Parent/admin/support copy is updated for changed behavior.
- Regression coverage exists for high-impact fixes.

### V8OPS-03 Live Learning Mobile Provider Incident Fixes

Acceptance criteria:

- Learning, AI/provider, mobile, notification, curriculum, teacher, recommendation, and parent progress issues are fixed or explicitly deferred.
- Curriculum edits remain specially authorized.
- AI autonomy remains reviewed or policy-bound.
- Mobile/provider limitations have support and user fallback copy.

### V8OPS-04 Rollout Hotfix Release And Communication Evidence

Acceptance criteria:

- Fixes include tests, release notes, support macros, incident communications, rollback notes, and dashboard evidence.
- Release evidence links code SHA, deploy/build IDs, request IDs where applicable, timestamp, and owner.
- User-visible limitations are not hidden.
- Residual risks are owner-assigned with next action.

### VERIFY-90 v8.1 Rollout Operations Decision Gate

Acceptance criteria:

- Decision is continue rollout, hold, rollback, remediation, or growth readiness.
- Decision uses customer, support, revenue, learning, mobile, provider, incident, release, and acquisition evidence.
- Roadmap, requirements, state, milestone snapshots, and project summary are updated.
- v8.2 proceeds only if rollout evidence supports growth/acquisition evaluation.

## Out of Scope

- Paid marketing execution without separate approval.
- Broad expansion beyond rollout scope.
- Unreviewed AI autonomy.
- Hiding known limitations.
