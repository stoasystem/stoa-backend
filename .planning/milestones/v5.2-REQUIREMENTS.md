# Requirements: v5.2 Adaptive Sequencing And Warehouse Analytics

**Milestone:** v5.2
**Status:** Complete
**Created:** 2026-06-14

## Goal

Turn STOA's adaptive memory, reviewed assignment, curriculum analytics, and curriculum product readiness foundations into a deeper adaptive sequencing and warehouse-backed analytics capability. v5.2 focuses on sequencing contracts, recommendation/candidate ranking, assignment outcome loops, analytics export/readiness, and operator dashboards.

This is an internal development milestone. Prioritize functional learning product capability and analytics readiness. Keep checks focused on recommendation correctness, assignment lifecycle behavior, aggregate analytics quality, and operator visibility; avoid broad unrelated security/compliance work.

## Requirements

### ADAPTWARE-01 Adaptive Sequencing And Warehouse Analytics Contract

Implementers have a concrete v5.2 contract before sequencing or warehouse analytics work expands.

Acceptance criteria:

- Contract identifies backend, frontend, curriculum, tutor, analytics, and release ownership boundaries.
- Contract defines adaptive sequencing inputs from memory snapshots, curriculum progress, mistakes, AI drafts, assignments, content quality, and tutor review.
- Contract defines ranking, dedupe, freshness, explanation, and review gates for recommended next work.
- Contract defines warehouse analytics export/readiness boundaries, aggregate metrics, dashboards, and future BI handoff.
- Contract defines rollout states, release evidence, and deferred autonomous tutoring scope.

### ADAPTWARE-02 Adaptive Sequencing Recommendation Engine

Students/tutors can receive better next-work recommendations from multiple learning signals without enabling fully autonomous tutoring.

Acceptance criteria:

- Sequencing service combines memory, curriculum progress, assignment outcomes, mistakes, content-quality signals, and reviewed AI draft availability.
- Candidate generation supports curriculum exercises, reviewed AI drafts, remediation topics, and continuation lessons.
- Ranking produces stable rationale, confidence, freshness, source signals, and review-required flags.
- Duplicate or stale recommendations are suppressed when assignments are active, completed, archived, rolled back, or content is unpublished.
- Tests cover candidate generation, ranking, dedupe, stale signal handling, review gates, and role-visible response shape.

### ADAPTWARE-03 Assignment Outcome Feedback Loop

Assignment outcomes feed sequencing and curriculum/content analytics in a repeatable way.

Acceptance criteria:

- Assignment start/complete/skip/archive outcomes update sequencing signals and content-quality analytics consistently.
- Completion outcomes record correctness, attempt metadata, source content, topic/subject, and remediation signals.
- Skip/archive outcomes influence future ranking without permanently hiding useful remediation.
- Parent/tutor progress signals explain sequencing changes without exposing internal ranking details.
- Tests cover outcome recording, recommendation changes after outcomes, repeated completion idempotency, skip/archival behavior, and parent/tutor summary shape.

### ADAPTWARE-04 Warehouse Analytics Export And Operator Dashboards

Learning/curriculum analytics are ready for warehouse-backed reporting and actionable operator dashboards.

Acceptance criteria:

- Export contract defines event/metric schemas for learning memory, assignment outcomes, curriculum progress, content quality, support/billing join keys where relevant, and aggregation windows.
- Backend exposes or documents warehouse-ready export/readiness behavior without requiring a live warehouse during internal development.
- Admin analytics expose cohort progress, sequencing coverage, assignment outcomes, curriculum quality, stale content, and intervention opportunities.
- Data is aggregate/operator-focused by default, with drilldown scope explicitly documented.
- Tests or focused checks cover export schema shape, aggregate metrics, empty states, and dashboard response contracts.

### VERIFY-35 v5.2 Adaptive Sequencing Analytics Release Gate

v5.2 closes with adaptive sequencing and analytics readiness evidence.

Acceptance criteria:

- Focused backend/frontend contract checks pass or isolate documented pre-existing failures.
- Sequencing engine, outcome feedback loop, warehouse export/readiness, analytics dashboard contracts, and docs are verified.
- Requirements, roadmap, state, feature gap docs, and remaining-feature queue reflect completed v5.2 work.
- Final audit records rollout state: contract-ready, sequencing-ready, analytics-ready, warehouse-ready, blocked, or deferred.
- Next milestone recommendation is updated from the remaining feature queue.
