# Roadmap: v5.27 Pilot Remediation Product Fit And Reliability Hardening

**Status:** Planned
**Created:** 2026-07-07
**Prior milestone:** v5.26 Limited Pilot Execution And Outcome Evidence

## Goal

Turn v5.26 pilot evidence into focused product, reliability, support, AI, curriculum, and mobile fixes before any expansion.

## Function Purpose

- Fix the issues actual pilot users hit.
- Improve activation, learning usefulness, support resolution, mobile stability, and trust signals.
- Keep remediation evidence tied to pilot findings rather than speculative polish.

## Implementation Strategy

- Prioritize by pilot severity, frequency, and impact on learning/retention.
- Separate quick product fixes from larger architecture or provider changes.
- Add regression coverage for every high-severity pilot issue.
- Close with a remediation gate that proves expansion blockers are resolved or explicitly accepted.

## Phases

- [ ] **Phase 307: Pilot Issue Triage And Remediation Backlog** - Convert pilot issues into severity-ranked product, support, provider, AI, curriculum, mobile, and reliability work.
- [ ] **Phase 308: Account Billing Mobile And Notification Remediation** - Fix high-impact onboarding, entitlement, billing, mobile, and notification issues found in pilot.
- [ ] **Phase 309: Learning AI Curriculum And Teacher-Help Remediation** - Improve AI quality, curriculum placement, assignment relevance, teacher-help routing, and parent/student explanations.
- [ ] **Phase 310: Pilot Regression And Reliability Evidence** - Add focused regression tests, e2e/smoke evidence, dashboards, and support-safe remediation evidence.
- [ ] **Phase 311: v5.27 Remediation Release Gate** - Decide whether the pilot can continue/expand or needs another remediation cycle.
