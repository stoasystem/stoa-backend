# Plan 196-01: Define Teacher Dispatch And SLA Load Balancing Contract

## Goal

Create a concrete v5.5 contract for automatic teacher dispatch, load balancing, timeout reassignment, queue visibility, and SLA evidence.

## Work Items

1. Confirm existing teacher request, queue, takeover, reply, resolve, notification, and SLA behavior.
2. Define function purpose: automatic dispatch of human help requests, not automatic answering.
3. Define teacher/tutor availability, subject capability, role eligibility, load metrics, and pause/offline states.
4. Define dispatch states, claim metadata, timeout/reassignment behavior, and manual takeover compatibility.
5. Define candidate ranking inputs: subject match, active load, recent SLA, last dispatch, queue age, and escalation priority.
6. Define teacher queue, operator dashboard, student status, and notification handoff.
7. Define Phase 197 through Phase 200 implementation targets and verification evidence.

## Verification

- Contract maps every TEACHDISP-01 acceptance criterion to implementation targets.
- Existing manual queue/takeover/reply workflow remains compatible.
- Follow-up phases have concrete backend, frontend-contract, and focused-check targets.
