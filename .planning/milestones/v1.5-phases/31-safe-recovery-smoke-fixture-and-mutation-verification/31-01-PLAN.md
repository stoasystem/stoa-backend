# Plan 31-01: Safe Recovery Smoke Fixture and Mutation Verification

## Steps

1. Prepare temporary non-customer parent/student/report fixtures and private S3 report HTML artifacts.
2. Verify valid non-admin rejection with a temporary parent token.
3. Verify admin list/detail against the safe fixture and confirm metadata-only responses.
4. Run `generation_failed` retry smoke on the retry fixture.
5. Run single `email_failed` resend smoke on the resend fixture.
6. Run selected bulk resend smoke with one valid failed fixture and one missing target.
7. Confirm audit/status fields and action eligibility after each mutation.
8. Delete temporary Cognito users, DynamoDB records, and S3 artifacts.

## Safety Gates

- Do not use customer report IDs.
- Do not run mutation endpoints until the fixture IDs are documented.
- Stop if responses expose private artifact keys, public URLs, presigned URLs, or direct S3 URL markers.
- Stop if cleanup cannot delete fixture data.
