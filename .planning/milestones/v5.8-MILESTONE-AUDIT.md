## Implemented Behavior

- New registration uses Cognito `sign_up` and stores `pending_verification`.
- Registration returns no access token until verification is confirmed.
- Login refuses token return for profiles requiring verification.
- Confirm verification uses Cognito `confirm_sign_up` and activates the local profile.
- Resend verification uses Cognito `resend_confirmation_code` with cooldown/idempotency behavior.
- Expired codes update local state to `expired_verification`.
- Admin support can inspect bounded verification state by user ID.
- Login-code/passwordless request and confirm return explicit `deferred` policy responses and no token.

## Residual Risk

- Production deploy/live Cognito smoke was not run in this local release gate.
- Existing legacy `admin_marked_verified` users remain compatible and are treated as verified.
- Login-code/passwordless requires future Cognito custom auth trigger design before support.
- Full account operations visibility remains v5.9 scope.

## Handoff

v5.9 should use the new verification metadata and `GET /admin/account-verification/{user_id}` as the verification slice of broader parent/admin operations visibility.
