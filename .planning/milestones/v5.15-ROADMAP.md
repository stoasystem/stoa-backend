# Roadmap: v5.15 Usage, Quota, And Product Stability

**Status:** Planned
**Created:** 2026-07-05
**Prior milestone:** v5.14 Verification And Login Reliability

## Goal

Make usage accounting and core product stability trustworthy: real usage events, quota reconciliation, user/admin explanations, health checks, smoke tests, and regression gates for the flows that matter most.

## Why This Is A New Milestone

Usage and quota behavior are product-critical because they affect access, billing explanations, and support decisions. Prior milestones expanded the ledger contract, but the next stability milestone should verify real app behavior against the code and close gaps in observability, reconciliation, and smoke coverage.

## Product Purpose

- Students and parents can trust usage and remaining quota displays.
- Admin/support can explain why access was allowed, limited, or blocked.
- Ledger and aggregate counters can be reconciled after retries or partial failures.
- Core flows have smoke checks and health evidence before production changes.
- Internal development can move faster because failures are easier to localize.

## Implementation Strategy

- Audit real flow coverage instead of assuming all actions are metered.
- Define canonical usage actions and skip rules for paid, free, retry, failed, and admin-triggered flows.
- Reconcile ledger rows, counters, entitlements, and support summaries.
- Add health endpoints or smoke scripts for auth, paid access, curriculum reads, question submission, teacher help, and admin support surfaces.
- Keep observability support-safe: metadata and request IDs over raw student content.

## Workstreams

1. Usage-flow reality audit.
2. Ledger coverage and idempotency closure.
3. Quota reconciliation and support explanations.
4. Core health/smoke and regression gates.
5. Stability release evidence.

## Acceptance

- Major student usage flows emit intended usage records or documented skip decisions.
- Quota counters reconcile against ledger events with drift detection.
- Parent/admin support summaries explain usage without raw learning content.
- Health/smoke checks cover login, entitlement, curriculum read, question submit, teacher help, and admin support surfaces.
- Tests cover retry/idempotency, failed operations, entitlement mismatch, counter drift, support summaries, and smoke script behavior.

## Out Of Scope

- Full BI warehouse deployment.
- Long-term analytics modeling beyond operational usage reliability.
- External APM/vendor rollout unless explicitly approved.
