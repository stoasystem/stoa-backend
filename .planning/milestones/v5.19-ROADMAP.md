# Roadmap: v5.19 Native Mobile Push And Offline Client Implementation

**Status:** Planned
**Created:** 2026-07-05
**Prior milestone:** v5.18 Warehouse BI Observability And Product Analytics Activation

## Goal

Move from web/PWA readiness and native handoff contracts into a real native/mobile implementation path: mobile client shell, auth/session handling, parent/student core journeys, native push registration, offline/read-through behavior, localization QA, and app-release evidence.

## Why This Follows v5.18

The backend and web app now have product-readiness evidence, and v5.17/v5.18 should clarify provider activation and operational observability. Native/mobile work should start after those boundaries are clear so the app does not inherit ambiguous payment, notification, support, or analytics states.

## Product Purpose

- Students and parents can use the core STOA experience on mobile with native-grade reliability.
- Push notifications and offline/read-through behavior have concrete client implementation, not only backend contracts.
- Mobile release evidence covers auth, billing state, curriculum/practice, teacher help, notifications, account operations, and localization.

## Workstreams

1. Native/mobile stack and app shell decision.
2. Auth/session, account state, and secure token storage.
3. Student and parent core journeys.
4. Native push token registration, notification handling, and deep links.
5. Offline/read-through cache, localization/text-fit QA, and app-release gate.

## Acceptance

- Mobile app uses real backend APIs with no hidden demo fallback for production-like state.
- Auth/session handling is secure and compatible with Cognito/token refresh behavior.
- Parent/student core journeys work on mobile.
- Native push registration and notification deep links are implemented against backend contracts.
- Offline/read-through behavior is explicit and bounded.
- App-release evidence includes tests, screenshots, known limitations, and provider/app-store prerequisites.

## Out of Scope

- Rebuilding backend product flows already covered by web.
- Unreviewed AI tutoring autonomy.
- App-store live launch without explicit release approval.
