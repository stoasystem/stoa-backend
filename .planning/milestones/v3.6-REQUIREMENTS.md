# Requirements: v3.6 Full WebSocket Realtime Notifications

**Milestone:** v3.6
**Status:** Active
**Created:** 2026-06-08

## Goal

Turn the v3.5 in-product notification foundation into full WebSocket realtime notifications for core learning and operations workflows.

## Requirements

| Requirement | Phase | Status |
|-------------|-------|--------|
| WS-01 Full WebSocket transport contract and infrastructure readiness | Phase 112 | Planned |
| WS-02 Backend WebSocket connection and event delivery | Phase 113 | Planned |
| UI-21 Realtime notification client and UX | Phase 114 | Planned |
| VERIFY-19 v3.6 functional release gate and realtime audit | Phase 115 | Planned |

## Product Build Priority

This milestone implements realtime product behavior on top of the existing notification foundation: authenticated connection, event fanout, client UX, and fallback behavior. It intentionally excludes native push and email digests.

## Out Of Scope

- Native push notifications.
- Email notification digests.
- Payment-provider implementation.
- Automatic exercise generation.
