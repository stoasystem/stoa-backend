# Requirements: v2.8 CDK-Managed Immutable Evidence Storage Deployment

**Milestone:** v2.8
**Status:** Complete
**Created:** 2026-06-07

## Goal

Deploy and enable CDK-managed immutable evidence storage for report operations retention manifests, then prove full metadata-only immutable manifest object persistence in production without exposing private artifacts, deleting audit rows, or mutating customer report artifacts.

## Requirements

| Requirement | Phase | Status |
|-------------|-------|--------|
| IMSTORE-01 Immutable evidence storage CDK design and deploy readiness | Phase 79 | Complete |
| IMSTORE-02 CDK-managed immutable evidence storage resource | Phase 80 | Complete |

| VERIFY-11 v2.8 release gate and live immutable storage verification | Phase 82 | Complete |

## Acceptance Summary

- CDK is the only approved source of truth for immutable storage resources, Lambda configuration, and IAM permissions.
- Backend immutable persistence remains fail-closed until CDK-injected configuration exists.
- Persisted immutable objects are metadata-only and digest-backed.
- Production smoke uses only approved metadata-only evidence or a named non-customer safe fixture.
- Release evidence proves no private artifact exposure, no audit deletion, no customer report mutation, and no external write.

## Future Requirements

## Out of Scope

- Manual AWS console changes.
- Deleting existing audit rows.
- Retaining raw report artifacts or private storage identifiers in committed evidence.
- Direct third-party support-system writes.
- Customer report artifact mutation during smoke.
- Claiming broad regulatory compliance beyond the specific CDK-managed immutable storage behavior that release evidence proves.
