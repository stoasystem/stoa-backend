## Audit Findings

### Finding 1: Backend deploy did not update scheduled Lambda

Initial integration audit found `.github/workflows/deploy.yml` built a Lambda package but updated only `stoa-api`. That would leave `stoa-weekly-report` stale after backend deployments.

Resolution:

- Fixed in `238eed9` (`ci: deploy weekly report lambda`).
- Backend deploy workflow now loops over `stoa-api` and `stoa-weekly-report` for both `update-function-code` and `function-updated` wait.

Status: closed.

### Finding 2: Local `dist/` is empty because it is a build artifact

`/Users/zhdeng/stoa-backend/dist` is gitignored and currently empty locally. CDK uses `../stoa-backend/dist` as the Lambda asset path, so local CDK deploys require building `dist` first.

Resolution/evidence:

- This is an operational precondition rather than a shipped source gap.
- Backend CI builds `dist` before zipping and now updates both Lambda functions.
- Infra CI checks out `stoa-backend` and builds `stoa-backend/dist` before `cdk diff` and `cdk deploy`.
- Local CDK users must run the same backend packaging step before CDK deploy.

Status: accepted operational note.
