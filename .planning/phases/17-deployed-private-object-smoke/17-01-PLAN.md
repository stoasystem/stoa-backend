# Plan 17-01: Weekly Lambda Private Artifact Smoke

**Phase:** 17 - Deployed Private-Object Smoke
**Status:** Planned
**Requirements:** SMOKE-01, SMOKE-02, SMOKE-03, SMOKE-04, SMOKE-05

## Objective

Add a narrow weekly report Lambda smoke event that writes and reads a deterministic private JSON artifact through Lambda IAM, without exposing S3 through any public route.

## Tasks

1. Add smoke support to `report_artifact_service`.
   - Build deterministic smoke key from canonical artifact helper.
   - `put_object` a small JSON payload with `application/json`.
   - `get_object` the same key and verify an expected marker.
   - Return only metadata: status, bucket, key, content type, readback success, and cleanup decision.

2. Route weekly Lambda handler smoke events.
   - In `weekly_reports.handler`, detect `event["job"] == "report_artifact_s3_smoke"`.
   - Run the smoke helper and return its result.
   - Keep scheduled weekly report behavior unchanged for all other events.

3. Add tests.
   - Handler invokes smoke path and does not run the weekly report job.
   - Smoke writes and reads exactly one canonical JSON object under `weekly-reports/`.
   - Smoke output omits object content and does not include public URL fields.

4. Record invocation guidance/evidence.
   - Add phase verification noting local fake-client proof and deployed invocation still pending if AWS CLI is unavailable.

## Verification

- `uv run pytest`
- `git diff --check`

## Acceptance Criteria

- SMOKE-01: smoke is Lambda-event only.
- SMOKE-02/03: smoke writes and immediately reads deterministic private JSON.
- SMOKE-04: output records metadata and readback success without content.
- SMOKE-05: smoke uses `PutObject`/`GetObject`, not public URL/list/log access.
