## Goal

Add provider-gated email digest and push delivery behavior with durable preferences, redacted evidence, token readiness, and focused tests.

3. Add provider readiness helpers for email and push channels.

5. Add push token registration/revocation helpers and push event delivery attempts with missing-token, opt-out, provider-failure, and success states.
6. Extend notification routes and response models for manual digest send and push token registration/revocation.
7. Add focused tests covering configured/unconfigured providers, preference opt-out, digest selection, push token missing, provider failure, and successful fixture sends.
