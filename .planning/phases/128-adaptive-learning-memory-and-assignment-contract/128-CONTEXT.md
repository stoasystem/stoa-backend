# Phase 128 Context: Adaptive Learning Memory And Assignment Contract

**Milestone:** v4.0 Adaptive Learning Memory And Assignment
**Requirement:** ADAPT-01
**Status:** Planned

## Why This Phase Exists

`stoa_docs` Phase 2 calls for personalized learning memory. The product now has subject/topic seeds, curriculum progress, AI teacher drafts, weekly report signals, and parent/tutor visibility. Phase 128 defines how these inputs become durable memory and reviewed assignment workflows before backend/UI implementation.

## Product Scope

- Durable memory snapshots by student, subject, and topic.
- Inputs from questions, feedback, tutor replies, curriculum progress, exercise attempts, AI exercise drafts, and weekly reports.
- Next-practice recommendations.
- Reviewed assignment lifecycle from draft/recommended to assigned/completed/skipped.
- Student/tutor/parent visibility and stale evidence behavior.

## Completion Criteria

Phase 128 completes when the memory fields, source inputs, assignment lifecycle, recommendation boundaries, visibility rules, stale-data behavior, and functional verification checklist are written and internally consistent.
