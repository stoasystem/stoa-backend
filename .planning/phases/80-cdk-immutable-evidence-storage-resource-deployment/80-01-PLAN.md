# Phase 80 Plan: CDK Immutable Evidence Storage Resource Deployment

**Status:** Complete
**Requirement:** IMSTORE-02

## Objective

Define, synthesize, deploy, and verify the CDK-managed immutable evidence storage resource and API Lambda runtime configuration needed for backend immutable manifest persistence.

## Steps

1. Add a retained Object Lock-enabled immutable evidence bucket in `StorageStack`.
2. Export the bucket name, approved prefix, retention mode, and retention days.
3. Inject immutable storage environment variables into `stoa-api` only.
4. Grant API Lambda `s3:GetObject` and `s3:PutObject` on the immutable prefix only.
5. Verify synth and production diff before deployment.
6. Push infra commit and wait for the normal GitHub Actions CDK deployment.
7. Verify live CloudFormation outputs, S3 Object Lock/versioning/encryption/logging/public access, Lambda env vars, and API IAM permissions.

## Non-Goals

- Backend behavior changes beyond consuming CDK-injected settings.
- Live immutable manifest persistence smoke; this remains Phase 81/82 scope after deployment is proven.
- Legal/compliance approval of the final retention period or legal hold procedure.
