<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Parent JWT to report routes | Authenticated caller supplies child id and optional week; route must verify local parent-child ownership before report reads. |
| Report repository to parent response | Stored report content can be returned only when parent id and student id match the verified route context. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03-S | Spoofing | Report route dependency | mitigate | Use `Depends(require_role("parent"))` on normal `/parents/me/...` report routes. |
| T-03-03-I | Information Disclosure | `_latest_report_for_child` and week route | mitigate | Verify owned child before reads, then verify `report.student_id == child_id` before available state. |
| T-03-03-T | Tampering | `week` path parameter | mitigate | Treat week as lookup key only; report output remains gated by verified parent id and child id. |
| T-03-03-R | Repudiation | Missing report response | accept | Endpoint is read-only and returns deterministic missing state without mutating data. |
| T-03-03-SC | Tampering | npm/pip/cargo installs | accept | No package-manager installs are part of this plan. |
</threat_model>
