| Boundary | Description |

| Parent JWT to `/parents/me/children/{child_id}/history` | Authenticated caller supplies `child_id` and `limit`; route must verify local parent-child ownership and bound limit. |
| Event aggregation to response | Multiple raw DynamoDB record types are normalized into a parent-visible timeline. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-02-S | Spoofing | History route dependency | mitigate | Use `Depends(require_role("parent"))` and Phase 2 local parent profile resolution. |
| T-03-02-I | Information Disclosure | History source reads | mitigate | Call `_get_owned_child_profile` before question, practice, conversation, or report reads. |
| T-03-02-T | Tampering | `limit` query parameter | mitigate | Validate with FastAPI `Query` bounds and never pass unbounded limits to helpers. |
| T-03-02-R | Repudiation | Event normalization | accept | This endpoint reads existing records and does not mutate state. |
| T-03-02-SC | Tampering | npm/pip/cargo installs | accept | No package-manager installs are part of this plan. |
</threat_model>
