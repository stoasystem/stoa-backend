# Plan 20-01: Scope Report Artifact Lambda IAM to weekly-reports

**Phase:** 20 - Prefix-Scoped Report Artifact IAM
**Status:** Planned
**Requirements:** IAM-01, IAM-02, IAM-03, IAM-04

## Objective

Replace broad reports bucket read/write grants with prefix-scoped object permissions for both report-capable Lambdas, then verify deployed IAM and live report artifact smoke still work.

## Tasks

1. Update CDK Lambda reports bucket permissions.
   - Remove broad `reports_bucket.grant_read_write(...)` for `self.api_function`.
   - Remove broad `reports_bucket.grant_read_write(...)` for `self.weekly_report_function`.
   - Add a local helper that grants only required report artifact object actions on `reports_bucket.arn_for_objects("weekly-reports/*")`.
   - Preserve `images_bucket.grant_read_write(self.api_function)`.

2. Generate local CDK/template evidence.
   - Run CDK synth for `StoaApiStack`.
   - Confirm synthesized IAM policy resources reference `weekly-reports/*` for reports bucket object actions.
   - Confirm reports bucket `*` object resources are removed from Lambda policies.
   - Confirm images bucket permissions remain unchanged.

3. Generate diff/deploy evidence.
   - Run `cdk diff StoaApiStack --profile stoa`.
   - Deploy `StoaApiStack` if the diff is limited to expected IAM policy changes plus known Lambda asset hash drift.
   - Query live IAM inline policies for both Lambda roles after deployment.

4. Verify runtime behavior.
   - Run focused backend tests for report artifact helper and weekly report smoke routing.
   - Invoke `stoa-weekly-report` with `{"job":"report_artifact_s3_smoke"}` and confirm it writes/reads under `weekly-reports/`.
   - Confirm API image bucket permissions remain present in synth/live IAM evidence.

5. Write execution artifacts.
   - Create `20-01-SUMMARY.md`.
   - Create `20-VERIFICATION.md`.
   - Run code review and record `20-REVIEW.md`.
   - Update `.planning/REQUIREMENTS.md`, `.planning/ROADMAP.md`, `.planning/STATE.md`, and `.planning/PROJECT.md`.
