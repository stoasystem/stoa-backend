<objective>
Regenerate Phase 473 validation and evidence on one final SHA, prove V9PRIV-02 restored and V9PRIV-01/03 retained, and preserve honest external NOT RUN boundaries.

## Artifacts this phase produces

- Byte-stable checked authorization inventory for all current routes.
- A finding matrix for CR-001 and WR-001..WR-005, a D-01..D-22 matrix, and V9PRIV-01..03 requirement matrix with exact source/test evidence.
- Immutable `testedSourceSha`, command, exit status, test count, inventory/artifact digests, and private-canary denylist results.
- Updated Nyquist validation rows for Plans 473-08..11.
- `phase-473-evidence-manifest.json` containing the tested source SHA plus SHA-256 digests and byte sizes for evidence and VALIDATION.
- Executor-created `473-11-SUMMARY.md` records the resulting evidence-docs commit SHA only after that commit exists.
- Explicit NOT RUN entries for real S3 version/chunk/promotion behavior, deployed cleanup schedule/IaC, and deployed log capture unless independently approved and executed.

<threat_model>
The final gate models TOCTOU replacement and concurrent completion, exact/mismatched/concurrent replay, transaction and quota/ref double effects, content/provider log disclosure, hidden-resource/error-oracle behavior, and provider/repository/AI failures. It treats dirty-tree, moving-SHA, and self-reference as threats: implementation plus inventory are committed first and every PASS derives from immutable clean `testedSourceSha`. Evidence, VALIDATION, and their manifest may contain only that tested source identity plus non-self-referential artifact/test digests; none may predict or embed the commit SHA that contains them. The executor records the resulting docs commit SHA afterward in `473-11-SUMMARY.md`. Immutable coordinate/content canaries remain prohibited and external proof stays honest NOT RUN. All named local gates block closure.
</threat_model>
