# Plan 473-26: Exact teacher course/class answer authorization

<objective>
Narrow privileged pre-attempt answer access to exact current teacher course-and-class assignments while retaining read-only admin access and uniform non-enumerating denials per D-21 and D-22.

Purpose: Prevent broad subject/grade overlap or stale assignment state from authorizing answer material.
Output: Fresh exact-scope policy, load-once direct resolver, and exhaustive authorization matrix.
</objective>

## Artifacts this phase produces

- Wave-0 cross-class/revoked/expired/overlap authorization tests.
- Exact course+class mandatory scope evaluation and fresh assignment validation.
- Load-once version/hash-validated challenge authorization with read-only inventory contract.

<must_haves>
Truths: teacher access requires exact current course and class; subject/grade never broaden; admin is read-only; all others are concealed. Artifacts: policy, resolver, repository read, and tests. Key links: canonical challenge to ResourceRef, assignment to policy, and policy to answer projection.
</must_haves>

<threat_model>
ASVS L1 assets are correct answers, explanations, feedback, challenge existence, teacher assignment scope, admin privilege, and authorization audit. Trust boundaries are opaque challenge ID to canonical loaded value, assignment repository to fresh facts, ResourceRef coordinates to policy, and policy result to route response. High-severity blockers are subject/grade-only authorization, cross-class access, stale/revoked/expired assignment, request-controlled course/class, scan ambiguity, admin mutation, parent/student enumeration, duplicate load/TOCTOU, or raw denial detail. Mitigate with Plan 25 direct resolver, exact course+class mandatory facts, fresh account/assignment reads, load-once authorization, read-only route inventory, concealed errors, and full matrix/audit tests.
</threat_model>
