# Plan 473-02: Upload intent, validation, quota and lifecycle core

<objective>
Replace key-returning presigned PUT uploads with owner-recorded, size-constrained presigned POST intents and authoritative post-upload validation, backed by conditional repository transitions and storage quota primitives.
</objective>

## Artifacts this phase produces

- `DetectedFile`, `ValidationFailure`, `validate_uploaded_file`, `validate_image`, `validate_pdf`, `validate_ooxml`, and `validate_text` in `file_validation_service.py`.
- Upload/attachment/storage key helpers and conditional operations in `attachment_repo.py`, including `create_upload_intent`, `get_upload_intent`, `begin_validation`, `mark_validated`, `mark_invalid`, `release_validation`, `get_attachment`, `get_storage_usage`, and transaction builders.
- `create_upload_intent`, `finalize_upload`, `resolve_owned_upload`, `resolve_owned_attachment`, and `storage_limit_for_entitlement` in `attachment_service.py`.
- Presigned POST response contract and finalize route under `/files`.

<must_haves>
Truths: D-01..D-09 and D-11/D-13..D-17 are enforced by byte/state/transaction boundaries. Artifacts: validators, repository, service and migrated file routes. Key links: clients never learn keys; only `validated` owner records can proceed to association plans.
</must_haves>

<threat_model>
ASVS L1 assets: uploaded bytes, private object coordinates, storage quota, intent state and provider diagnostics. Trust boundaries: declared filename/MIME to signed policy; direct S3 upload to finalize; S3 metadata/body to validator; conditional state to API response. Threats: oversize upload, MIME/magic mismatch, polyglot or corrupt content, OOXML traversal/ZIP bomb/encryption, image decompression bomb, race between finalizers, expired intent reuse, quota spoofing, foreign ID enumeration and raw botocore error leakage. Mitigations: exact POST policy, HeadObject, bounded stream, Pillow verify/edge/bomb checks, ZIP member/count/ratio/path ceilings, strict UTF-8, conditional versions/status, server clock, Actor owner comparison and category-only errors. Verification: valid/boundary/one-byte-over fixtures, concurrent conditions, fake S3 exceptions and canary scans. Residual risk: malware scanning beyond structural validation is not available; unsupported/active/encrypted document content is rejected and infrastructure scanning can be added later without changing the contract.
</threat_model>
