# Plan 473-38: Authoritative private delivery and sealed global exception

<objective>
Close CR-02 by making private digest, push, and WebSocket delivery fail closed without authoritative owner/generation, permitting ownerless delivery only for a persisted sealed global-nonprivate row, and routing every permitted effect through Plan 473-37 intent/lease fencing per D-10, D-16, and D-17.

Purpose: Prevent missing, malformed, stale, or legacy notification metadata from bypassing the permanent account fence and delivering private content after account deletion begins.
Output: Strong delivery-source resolution, closed legacy joins, sealed global classification, unified channel fencing, stable fail-closed outcomes, and deletion-race tests.
</objective>

## Artifacts this phase produces

- New `AuthoritativeDeliveryOwnership` symbol with exact `private_owner` and `global_nonprivate` variants.
- New `LEGACY_NOTIFICATION_OWNER_RESOLVERS` closed registry mapping each private target family to a strongly consistent base-row loader; event metadata, recipient, actor, and role are never resolver authority.
- New `GLOBAL_NONPRIVATE_DELIVERY_CONTRACTS` closed allowlist and persisted fields `owner_classification`, `classification_contract`, `classification_digest`, and `event_version`.
- New repository/service operations `load_delivery_event_strong`, `resolve_legacy_delivery_owner`, `resolve_delivery_ownership`, and `run_authoritative_delivery`.
- New stable internal outcomes `delivery_owner_unresolved`, `delivery_classification_invalid`, `delivery_scope_mismatch`, `canceled_account_deletion`, and `provider_acceptance_unknown`.
- New `tests/test_phase473_private_delivery_fencing.py` with legacy metadata, sealed-global, mixed digest, stale generation, and deletion-race cases.

<must_haves>
Truths: authoritative persisted ownership is mandatory, legacy joins are strong/closed, only sealed global rows are ownerless, invalid scope makes zero effects, and all permitted effects use intents. Artifacts: ownership contract, resolver/seal registries, unified adapter, statuses, and tests. Key links: strong event to owner, target to join, scope to intent, and fence change to zero provider mutation.
</must_haves>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| caller/queued event -> persisted event | Worker input may omit, forge, or contain stale owner-generation metadata. |
| legacy event -> authoritative target | Old rows require a strong target join; metadata/recipient/actor are untrusted ownership hints. |
| ownerless row -> global classification | An inferred or weakly persisted global label could become a private-delivery bypass. |
| authoritative scope -> channel provider | Deletion may begin after resolution or claim but before provider mutation. |
| multi-event digest/fanout -> one operation | Mixed owners/generations or partial provider effects must not escape fencing. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-473-38-01 | Spoofing | event ownership | mitigate | Strong-load canonical event and resolve private target through a closed authoritative join. |
| T-473-38-02 | Information disclosure | direct fallback | mitigate | Remove all direct provider fallbacks; unresolved/malformed/stale scope produces zero effect. |
| T-473-38-03 | Elevation of privilege | global classification | mitigate | Require persisted sealed contract/digest and strict nonprivate payload allowlist; no event-type inference. |
| T-473-38-04 | Tampering | mixed digest batch | mitigate | Require every digest item to share the exact scope/generation/classification and bind the canonical event set digest. |
| T-473-38-05 | Repudiation | provider race | mitigate | Stable logical operation, Plan 473-37 claim/begin CAS, immediate final fence check, and ambiguity terminalization. |
| T-473-38-SC | Tampering | package installs | mitigate | No package-manager install is planned; retain the package-legitimacy gate if scope changes. |
</threat_model>
