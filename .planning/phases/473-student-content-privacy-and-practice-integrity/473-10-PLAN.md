# Plan 473-10: Conversation replay convergence and private telemetry

<objective>
Close WR-005 and WR-002 by making regular and streaming conversation sends one replayable command and replacing content/provider logging with category/class/size/correlation-only telemetry across AI, OCR-fed, title, and conversation paths.
</objective>

## Artifacts this phase produces

- Required `idempotencyKey` on `SendMessageRequest`, bounded to 64 safe characters.
- SHA-256 fingerprint input `stoa.conversation.send.v1 || u32be(content_utf8_length) || exact_content_utf8 || u32be(reference_count) || repeated(type_byte || u32be(id_utf8_length) || id_utf8)` where type byte is `0x01` upload or `0x02` saved attachment; no trimming, Unicode normalization, delimiter ambiguity, or private content storage is allowed.
- `MESSAGE_COMMAND#{conversation_id}#{idempotency_key}` state with deterministic message IDs, step flags, AI lease, and serialized original safe response.
- `CHAT_QUOTA_OP#{command_id}` conditional row written in the same transaction as command creation and the daily counter increment.
- AI lease fields `leaseOwner`, `claimedAt`, `expiresAt`, and `attempt` with closed `message_committed -> ai_running -> completed|terminal_failed` transitions, 120-second expiry, at most two takeover attempts, and bounded duplicate polling.
- Stable `message_idempotency_conflict` and `message_in_progress` contracts with bounded retry actions.
- One shared regular/stream command executor and category-only logging helper.
- Two-stage route orchestration replacing the eager `_message_attachment_dependency`: Stage A command lookup/replay, Stage B attachment resolution plus command/quota claim only for an absent command.
- Lost-response, mismatched-reuse, concurrent duplicate, and private-canary tests.

<threat_model>
TOCTOU/concurrent-write threats include two requests claiming the same key, quota changing after attachment preflight, attachment conditions racing, and two AI callers. Stage A read-only command lookup occurs before any attachment dependency; exact replay therefore survives consumed uploads. Only an absent command reaches Stage B owner/state resolution, followed by atomic command/quota creation; a concurrent claim loser re-reads instead of repeating resolution/effects. Deterministic IDs, the Plan 473-09 mapping, and a fenced AI lease select one winner. Replay threats include response loss, exact retries, attachment reordering, Unicode/delimiter ambiguity, and mismatched reuse; the versioned length-prefixed fingerprint returns the original or stable conflict before attachment existence can become an oracle. Log disclosure remains category/class/size/correlation-only. Quota/ref double effects are prevented because command, daily counter, and conditional quota-operation row commit atomically and attachment transactions use deterministic IDs. AI duplicates poll at most 20x50 ms; an unexpired lease cannot be stolen, expired takeover is bounded twice, and exhaustion becomes terminal. External provider crash limitations remain explicit.
</threat_model>
