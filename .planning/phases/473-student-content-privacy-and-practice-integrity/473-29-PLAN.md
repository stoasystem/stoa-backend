# Plan 473-29: Durable deny-first account deletion and primary private-content purge

<objective>
Implement the real D-10 self-account deletion command and resumable reconciler, with a permanent deny-first identity fence and exhaustive question/OCR plus attachment lifecycle branches.

Purpose: Make account deletion an executable, restartable privacy guarantee rather than an injected hook, while preserving only the minimum lifecycle/accounting facts required for reconciliation.
Output: Self-delete route, durable command/state, permanent identity fence, primary purge branches, background job, and adversarial tests.
</objective>

## Artifacts this phase produces

- A durable account-deletion aggregate with an exact required branch registry and independent progress per branch.
- A strongly paginated question/OCR private-field scrubber and transactionally fenced question writes.
- A deny-first self-delete route and restartable reconciler/job consuming the Plan 473-23 attachment branch.

<must_haves>
Truths: acceptance follows the deny fence; private writes cannot cross it; each purge branch resumes independently; only exhaustive quiescence permits finalization; the terminal fence is permanent. Artifacts: deletion repository/service/job/route, question scrub/fences, and tests. Key links: route to command, fence to every write, branch cursors to reconciler, and quiescence to terminal tombstone.
</must_haves>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| authenticated student -> self-delete route | An active identity requests an irreversible lifecycle transition. |
| account fence -> question and attachment writers | Foreground and background mutations race deletion and must be transactionally refused. |
| repository/provider page -> durable branch state | Partial pages, malformed continuations, failures, and lost responses must not erase work or imply completion. |
| branch result -> aggregate finalizer | A single complete/count result cannot authorize account completion. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-473-29-01 | Elevation of privilege | `DELETE /auth/me` | mitigate | Resolve the current active subject only, conditionally fence that exact profile, and expose no arbitrary user ID. |
| T-473-29-02 | Tampering | question create/update paths | mitigate | Put every write in a same-table transaction with the matching account-fence generation and require existing rows on updates. |
| T-473-29-03 | Information disclosure | question/OCR rows and account profile | mitigate | Remove a closed private-field set and retain only an allowlisted tombstone/accounting projection. |
| T-473-29-04 | Denial of service | paginated purge/reconciler | mitigate | Persist independent cursors/debt, bound each job pass, retry idempotently, and require a zero-discovery quiescence pass. |
| T-473-29-05 | Repudiation | lost route/job responses | mitigate | Use immutable command ID/generation, conditional version transitions, and idempotent replay receipts. |
| T-473-29-SC | Tampering | package installs | mitigate | No package-manager install is planned; retain the standard package-legitimacy gate if scope changes. |
</threat_model>
