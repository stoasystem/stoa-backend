# Plan 473-29: Canonical deny-first account fence and primary content purge

<objective>
Implement D-10's real self-delete command on one permanent account fence, authorize post-fence retries without restoring active authority, and close the profile/question/OCR/teacher-session plus attachment/upload lifecycle races.

Purpose: Make deletion acceptance immediately deny access and make every existing primary-content writer share one transactional lifecycle truth before later plans add the remaining private-store branches.
Output: Canonical fence/command, replay-only route dependency, fenced writers, resumable primary branches, real continuation triggers, and adversarial tests.
</objective>

## Artifacts this phase produces

- A permanent `PK=USER#{user_id}`, `SK=ACCOUNT_FENCE` row with status, generation, command identity, conditional version, and terminal lifecycle facts.
- An immutable deletion command bound to verified issuer/subject and exact request fingerprint, separate from ordinary Actor authorization.
- Primary account/profile, question/OCR/teacher-session, and Plan 23 attachment branch implementations with independent cursor/debt/quiescence state.
- A route-after-commit worker trigger and a scheduled pending-command discovery handler.

<must_haves>
Truths: one permanent fence controls every writer; replay is command-only; primary private rows are exhaustively scrubbed; workers resume without a user session; only Plan 35 may seal/finalize the full registry. Artifacts: fence/command repositories, route dependency, fenced primary writers, purge handlers, triggers, and tests. Key links: verified subject to command, fence to transactions, triggers to durable claims, and branch progress to the later seal.
</must_haves>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| verified token -> deletion route | A signed token may identify a now-fenced subject but must not regain normal application authority. |
| canonical fence -> profile/question/attachment writers | Stale foreground and worker calls can otherwise repopulate content after the zero scan. |
| route commit -> worker trigger | A response or process can be lost after the command commits. |
| paginated rows/provider state -> branch progress | GSI gaps, malformed cursors, partial pages, and commit-then-raise can hide remaining work. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-473-29-01 | Elevation of privilege | delete replay dependency | mitigate | Bind immutable issuer/subject/fingerprint and expose only the same deletion receipt, never Actor/business capabilities. |
| T-473-29-02 | Tampering | account/profile/content writers | mitigate | Condition every mutation on the exact canonical fence key, active status, and generation in the same transaction. |
| T-473-29-03 | Information disclosure | question/OCR/session rows | mitigate | Base-scan all pages and REMOVE a closed private set to a strict noncontent tombstone allowlist. |
| T-473-29-04 | Denial of service | command continuation | mitigate | Persist pending commands, bounded leases/cursors/debt, route trigger, and scheduled discovery. |
| T-473-29-05 | Repudiation | lost route/provider/repository response | mitigate | Immutable command IDs, CAS transitions, exact debt identities, and idempotent replay receipts. |
| T-473-29-06 | Elevation of privilege | identity/public command/capability | mitigate | Terminalize old commands, revoke provider identity/session state, and scrub target scopes under the same permanent fence. |
| T-473-29-07 | Information disclosure | teacher SQS payload | mitigate | Send only opaque operation/question/generation coordinates and fence both producer and consumer; legacy payloads remain drain debt. |
| T-473-29-SC | Tampering | package installs | mitigate | No package-manager install is planned; retain the standard package-legitimacy gate if scope changes. |
</threat_model>
