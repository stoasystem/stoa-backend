# Plan 473-32: Conversation, message-command, and in-flight AI deletion closure

<objective>
Close account deletion for conversations and all duplicated message/command/note/help state, including regular/SSE parity, attachment associations, and assistant completions arriving after the fence.

Purpose: Prevent stored student/AI/teacher conversation text or command result JSON from surviving or being recreated by an in-flight lease after account closure begins.
Output: Complete writer/schema registry, same-fence mutations, restartable association-aware purge, and focused race tests.
</objective>

## Artifacts this phase produces

- A closed inventory for `CONV#{conversation_id}/CONV`, every message and `NOTE#` row, help lifecycle fields, `MESSAGE_COMMAND#`, chat quota operations, and chat usage-ledger metadata.
- Fenced repository transactions for conversation/message/command/AI lease completion paths shared by regular and SSE routes.
- An independent `conversation_messages` handler that drains commands, releases attachments, scrubs content, and proves later quiescence.

<must_haves>
Truths: all conversation copies are inventoried; all writers share the fence; in-flight completions cannot resurrect; associations release first; only noncontent accounting remains; quiescence is exhaustive. Artifacts: registry, fenced transactions, purge handler, and lower-bound tests. Key links: owner to fence, lease to recheck/completion, associations to release, and pages to branch state.
</must_haves>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| regular/SSE route -> repository transactions | Parallel implementations and legacy initial-message paths can bypass a shared command fence. |
| claimed command/Bedrock provider -> completion | An assistant result can return after deletion starts and write message/result duplicates. |
| conversation row -> attachment association | Scrubbing references before release can strand bytes or quota debt. |
| paginated mixed row families -> branch state | Conversation, messages, notes, commands, and usage copies can sit on different pages. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-473-32-01 | Information disclosure | messages/notes/commands | mitigate | Source-register and REMOVE text, result JSON, fingerprints, summaries, and private linkage to a strict tombstone. |
| T-473-32-02 | Tampering | AI lease completion | mitigate | Check the canonical generation at claim, pre-provider, renewal, and atomic assistant-message completion. |
| T-473-32-03 | Denial of service | attachment release | mitigate | Persist per-association Plan 22/23 results/debt and scrub only after exact release. |
| T-473-32-04 | Repudiation | lost command response | mitigate | Deterministic command IDs, conditional states, and idempotent canceled/completed reconciliation. |
| T-473-32-SC | Tampering | package installs | mitigate | No package-manager install is planned; retain the standard package-legitimacy gate if scope changes. |
</threat_model>
