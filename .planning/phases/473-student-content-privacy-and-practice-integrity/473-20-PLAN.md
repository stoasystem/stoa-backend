# Plan 473-20: Typed conversation command and durable usage lifecycle

<objective>
Replace boolean/exception ambiguity in the conversation command with an explicit durable state machine that atomically binds message effects, usage identity, and retry semantics per D-07, D-11, D-12, D-14, D-16, and D-17.

Purpose: Ensure retries, contention, deterministic rejection, lost responses, and midnight rollover converge to one message/usage outcome without false 429s or stuck charges.
Output: Typed repository dispositions, transactional usage staging, state-aware router behavior, and regular/SSE fault tests.
</objective>

## Artifacts this phase produces

- Wave-0 tests for every command stage below the adapter boundary.
- A typed command transition contract with durable request/quota/usage/replay facts.
- Removal of the `initialMessage` bypass from conversation creation.

<must_haves>
Truths: each command stage has an explicit outcome; deterministic failure is compensated or terminal; usage is not a separate fallible post-effect. Artifacts: enums/results, durable command fields, transaction changes, and endpoint parity tests. Key links: claim to bind, bind to usage, state to polling, and both transports to one executor.
</must_haves>

<threat_model>
ASVS L1 assets are student messages, attachment ownership, daily quota, usage ledger, idempotency fingerprint, assistant result, and private diagnostics. Trust boundaries are request to command claim, DynamoDB transaction responses to typed outcomes, usage data to durable event, and command state to regular/SSE responses. High-severity blockers are false 429 on contention, quota charge without a recoverable command, deterministic bind error masked as in-progress, duplicate/missing usage, recomputed midnight period, bool collapse of retryable completion, legacy initialMessage bypass, raw adapter failure, or regular/SSE divergence. Mitigate with conditional typed transactions, persistent identity facts, compensating terminal transition, closed adapters, state-aware polling, low-level fault injection, and identical endpoint executor wiring.
</threat_model>
