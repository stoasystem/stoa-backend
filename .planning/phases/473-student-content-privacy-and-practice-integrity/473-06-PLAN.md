# Plan 473-06: Scoped teacher and global admin answer access

<objective>
Implement the locked privileged answer rule as one narrow central policy: assigned teachers may read answers only for their current curriculum scope, admins may read all answers, and no role receives curriculum mutation as a side effect.
</objective>

## Artifacts this phase produces

- `ResourceType.CURRICULUM_ANSWER` and `AuthorizationPurpose.CURRICULUM_ANSWER_READ`.
- `CurriculumAnswerAuthorizationFacts`, policy evaluation branch and `authorized_curriculum_answer_dependency`.
- `question_repo.get_teacher_curriculum_assignment` (or equivalent current assignment scope loader).
- `GET /practice/curriculum/challenges/{challenge_id}/answer` with `PrivilegedPracticeAnswer` response.

<must_haves>
Truths: D-21/D-22 are exact central authorization behavior. Artifacts: resource/purpose/facts/dependency/endpoint. Key links: route inventory derives metadata from the executable dependency, and curriculum mutation capabilities remain unchanged.
</must_haves>

<threat_model>
ASVS L1 assets: pre-submission answer keys/explanations and teacher scope. Trust boundaries: Actor role to narrow policy; challenge ID to curriculum metadata; teacher assignment row to scope match; policy decision to answer projection. Threats: any teacher global read, stale/disabled assignment, caller-supplied course/class scope, admin answer rule broadening support/mutation access, student/parent `includeAnswers`, guessed challenge enumeration or policy outage default allow. Mitigations: load-once challenge metadata, fresh current assignment, exact course/class/subject/grade/lesson scope matching, narrow admin policy branch, answer-only response model, executable route inventory and fail-closed outage. Verification: assigned/unassigned/stale/wrong-scope teacher matrix, admin positive, student/parent/anonymous negative, mutation capability negative and hidden random/real comparisons. Residual risk: Phase 475 owns assignment write consistency; this phase reads current rows fail-closed.
</threat_model>
