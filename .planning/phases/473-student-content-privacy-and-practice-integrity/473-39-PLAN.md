<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| changed source -> checked inventory | Regeneration must not self-approve an unsafe mutation. |
| persisted row -> parser/decision | Malformed lease, time, version, digest, or classification values can broaden authority. |
| inventory selector -> runtime evidence | A high-level test may pass without invoking the declared lower boundary. |
| read inventory -> private-write composition | Divergent IDs/digests/branch sets can leave mutations unobserved. |
| evidence artifact -> privacy boundary | Generated data must not contain payloads, identifiers, provider coordinates, or absolute paths. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-473-39-01 | Tampering | source seal | mitigate | Independently reviewed mutating-source digests plus semantic AST checks on every generation/check run. |
| T-473-39-02 | Elevation of privilege | persisted-read parser | mitigate | Closed typed parsing and malformed-input mutation cases for every authority-bearing field. |
| T-473-39-03 | Repudiation | runtime selector | mitigate | Exact pytest node plus lower fake target and observed call/condition assertion. |
| T-473-39-04 | Information disclosure | generated artifacts | mitigate | Source-relative metadata/digests/status only and fixed private/provider-coordinate canary denial. |
| T-473-39-05 | Denial of service | registry drift | mitigate | Deterministic generation twice, exact branch composition, and fail-closed missing/extra/duplicate rows. |
| T-473-39-SC | Tampering | package installs | mitigate | No package-manager install is planned; retain the package-legitimacy gate if scope changes. |
</threat_model>
