# Plan 473-27: Exhaustive untrusted-response dataflow and compositional fault gate

<objective>
Make every actual consumption of untrusted provider/repository/parser response data explicit, strictly parsed, source-bound, and executable across D-01 through D-22.

Purpose: Prevent an unsafe new field read or coercion from hiding inside an already registered operation or behind a high-level happy-path test.
Output: Deterministic dataflow/route inventories, AST taint gate, isolated mutation tests, and runtime lower-bound sentinels.
</objective>

## Artifacts this phase produces

- A checked registry whose unit is one untrusted response consumption dataflow.
- Static source validation that permits consumption only through registered strict parsers.
- Runtime sentinels proving each selector reaches its declared lower fake and rejects malformed fields.

<must_haves>
Truths: every consumed untrusted value is registered; parser placement and strictness are enforced; new reads fail inside old operations; runtime reaches the lower fake; regular/SSE remain separate. Artifacts: generator, checked JSON, mutation/runtime tests, and route inventory. Key links: call to parser, field to helper, dataflow to selector, and route to authorization contract.
</must_haves>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| provider/repository/parser client -> application | Remote or persistence responses can be malformed, partial, ambiguous, truncated, or coercion-hostile. |
| source code -> checked registry | New response aliases/field reads can evade an operation-family inventory. |
| registry selector -> executable test | A selector may exist without reaching the declared lower boundary. |
| test observations -> evidence | High-level monkeypatches can claim coverage without exercising strict parsing. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-473-27-01 | Tampering | response consumption | mitigate | AST-trace every tainted alias/field to an immediate named strict parser and reject raw operators. |
| T-473-27-02 | Spoofing | checked source digest | mitigate | Re-run semantic taint rules on `--check`; a regenerated digest cannot authorize an unsafe source shape. |
| T-473-27-03 | Repudiation | selector coverage | mitigate | Install the declared lower fake and require its runtime call counter plus malformed parser assertion. |
| T-473-27-04 | Information disclosure | inventories/test output | mitigate | Emit only source-relative structural metadata and scan for private/provider canaries. |
| T-473-27-05 | Denial of service | alias/dataflow analysis | mitigate | Bound intra-symbol analysis, reject unresolved dynamic aliases, and keep deterministic sorted output. |
| T-473-27-SC | Tampering | package installs | mitigate | No package-manager install is planned; retain the standard package-legitimacy gate if scope changes. |
</threat_model>
