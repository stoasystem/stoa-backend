# Plan 473-34: Notification, assistance, device, and realtime deletion closure

<objective>
Close every durable notification and teacher-assistance copy, revoke device/realtime credentials, and fence all pending digest/push/WebSocket delivery work—including role broadcasts whose student owner is only present in target metadata.

Purpose: Prevent private summaries or credentials from persisting in notification rows or being delivered after account deletion begins.
Output: Owner resolution, fenced writers/provider leases, exhaustive notification/device purge, external-delivery classification, and focused tests.
</objective>

## Artifacts this phase produces

- A source-backed schema/writer inventory for notification events/attempt arrays, assistance summaries, preferences, push tokens, and WebSocket connections.
- An owner/generation envelope and durable provider-send intent used by digest, push, and realtime fanout.
- An independent `notification_device_realtime` branch and external-delivery subledger consumed by Plan 35.

<must_haves>
Truths: derived events and assistance text are owner-resolved/scrubbed; device credentials are revoked; every writer/provider effect checks the fence; external accepted delivery is not overclaimed; role broadcasts are classified. Artifacts: registry, send lease, purge handler, and tests. Key links: target to owner, claim to provider, pages to branch, and receipts to Plan 35.
</must_haves>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| producer target/envelope -> notification owner | Broadcast rows may have no recipient while metadata still identifies a student/question. |
| delivery worker -> email/push/WebSocket provider | Claimed work may execute after deletion or return ambiguous acceptance. |
| account deletion -> device credentials | Push token references/hashes and connection endpoints remain usable until revoked/deleted. |
| mixed global scans -> branch result | One-page limits and filtered empties can omit events, attempts, tokens, summaries, or connections. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-473-34-01 | Information disclosure | events/assistance | mitigate | Resolve owner from authoritative target/envelope and REMOVE all private summary/metadata fields. |
| T-473-34-02 | Elevation of privilege | device/realtime credentials | mitigate | Fence registration/refresh and revoke/delete token references, hashes, endpoint IDs, and subscriptions. |
| T-473-34-03 | Repudiation | provider delivery | mitigate | Durable send intent/lease, immediate fence recheck, strict acceptance parsing, and unknown-state no-blind-retry. |
| T-473-34-04 | Denial of service | paginated scans | mitigate | Independent family cursors/debt, strict progress validation, and later zero scans. |
| T-473-34-SC | Tampering | package installs | mitigate | No package-manager install is planned; retain the standard package-legitimacy gate if scope changes. |
</threat_model>
