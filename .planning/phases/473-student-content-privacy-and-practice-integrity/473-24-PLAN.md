# Plan 473-24: Semantic document validation and parser resource isolation

<objective>
Replace raw-byte and filename heuristics with semantic passive-document validation and bounded isolated extraction for every D-01 through D-05/D-08/D-16/D-17 supported type.

Purpose: Prevent active-content, entity, hybrid-container, archive-bomb, and parser resource bypasses before student content reaches durable use or AI context.
Output: Shared semantic validators, an isolated parser worker, and adversarial admission/extraction tests.
</objective>

## Artifacts this phase produces

- Wave-0 hostile OOXML/PDF/XML/resource tests.
- Semantic OPC relationship/content-type/main-part validation used at admission and extraction.
- CPU/memory/time/output bounded parser isolation with closed error categories.

<must_haves>
Truths: file type is semantically proven; active/external/entity content is rejected regardless of spelling; parser allocation is bounded; extraction repeats validation. Artifacts: validator, worker, integration, and tests. Key links: admission to semantic package facts, immutable bytes to revalidation, and worker outcome to closed command failure.
</must_haves>

<threat_model>
ASVS L1 assets are student document bytes/text, parser process memory/CPU, AI prompt, attachment type, and private diagnostics. Trust boundaries are untrusted archive/PDF/XML bytes to validators, ZIP metadata to allocation, parser subprocess to application, and extracted text to AI context. High-severity blockers are extension-only OOXML acceptance, duplicate/case-collision path confusion, macro/ActiveX/OLE/embedding/external link, relationship spelling bypass, DTD/entity expansion, decompression/decoded allocation before limits, unsupported/encrypted container acceptance, parser hang/OOM, extraction bypass of admission, or raw content/parser errors. Mitigate with canonical OPC graphs, duplicate refusal, event-driven XML rejection, streaming budgets, isolated process limits/deadline, same-policy revalidation, stable categories, and no-content telemetry.
</threat_model>
