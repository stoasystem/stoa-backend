# Plan 473-33: Practice, adaptive learning, draft, and analytics deletion closure

<objective>
Close every practice and adaptive-learning private store, including raw answers, immutable answer-bearing receipts, copied AI drafts, learning memories, and per-student curriculum analytics contributions.

Purpose: Prevent student learning history and derived teaching content from surviving in secondary rows or anonymous-looking deterministic hashes after account deletion.
Output: Fenced writers, five exhaustive purge branches, exact analytics recomputation, accounting minimization, and focused tests.
</objective>

## Artifacts this phase produces

- Source registries for practice/progress/attempt/mistake, adaptive assignment/memory, AI teacher draft, usage metadata, and curriculum signal families.
- Same-table canonical-fence transactions for every current producer/update path.
- Independent branch state with family cursors, exact item/contribution debt, and later zero-discovery proof.

<must_haves>
Truths: raw practice/adaptive/draft/memory content is scrubbed; copied drafts are covered; deterministic analytics linkage is removed with exact aggregate reconciliation; accounting is noncontent and TTL-bound; all writers are fenced. Artifacts: five handlers, fenced writers, analytics owner/recompute protocol, and tests. Key links: routes to fence, drafts to assignments, signal to aggregate, and cursors to Plan 35.
</must_haves>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| practice/adaptive route -> repository | Direct puts/updates can race the permanent deletion fence. |
| AI teacher draft -> accepted assignment | Private generated text is duplicated into `items`, answer keys, rationale, and notes. |
| student behavior -> curriculum analytics | An unsalted deterministic `studentHash` remains owner-resolvable in globally keyed rows. |
| item pages/aggregate metrics -> branch result | Partial deletion or double decrement can corrupt analytics or falsely imply purge. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-473-33-01 | Information disclosure | practice/adaptive rows | mitigate | REMOVE the complete source-registered private field sets to strict tombstones. |
| T-473-33-02 | Tampering | writers/updates | mitigate | Canonical fence ConditionCheck and existing owner row on every put/update transaction. |
| T-473-33-03 | Information disclosure | curriculum studentHash | mitigate | Stop deterministic linkage, maintain a fenced owner manifest, delete signals, and reverse contributions exactly once. |
| T-473-33-04 | Denial of service | aggregate reconciliation | mitigate | Durable per-signal operation ID/debt, conditional decrement/recompute, and later authoritative verification. |
| T-473-33-SC | Tampering | package installs | mitigate | No package-manager install is planned; retain the standard package-legitimacy gate if scope changes. |
</threat_model>
