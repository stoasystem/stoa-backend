# Plan 473-30: Derived private-copy purge and account-deletion finalization

<objective>
Complete D-10 by scrubbing every durable question-derived copy and report artifact, fencing all recreation paths, and allowing the Plan 473-29 aggregate to finalize only from exact branch-level absence proof.

Purpose: Ensure deleting source questions does not leave private summaries, drafts, memories, reports, or provider artifacts elsewhere in the backend.
Output: Derived-content purge handlers, fenced writers, exact report artifact cleanup, aggregate finalization, and adversarial tests.
</objective>

## Artifacts this phase produces

- Independent purge branches for moderation, assistance summaries, AI teacher drafts, learning memory, report rows, and report S3 artifacts.
- Transactional account-fence checks on every creator/updater for those copies.
- Exact nine-branch aggregate completion proof with privacy canaries and restart/lost-response coverage.

<must_haves>
Truths: no derived private copy remains or can be recreated; provider artifacts have verified absence; retained rows are minimal tombstones; aggregate finalization is exact and permanent. Artifacts: repository branches/fences, report artifact debt, service/job registration, and tests. Key links: fence to writers, owner discovery to scrub, report coordinates to absence proof, and all branches to final state.
</must_haves>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| source question -> derived repositories | Private content is copied into moderation, notification, teacher-tool, learning, and report rows. |
| deletion fence -> asynchronous writers | Existing workers may attempt to create/update derived copies after deletion begins. |
| report row -> object storage | Private report bytes and provider coordinates leave DynamoDB for S3. |
| branch registry -> account finalizer | Incomplete or omitted derived branches must never be interpreted as deletion completion. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-473-30-01 | Information disclosure | derived rows | mitigate | Remove an enumerated private-field set and assert canary absence in raw repository rows. |
| T-473-30-02 | Tampering | derived repository writers | mitigate | Condition each put/update on the current active account-fence generation and require existing rows for updates. |
| T-473-30-03 | Repudiation | S3 report deletion | mitigate | Persist exact-object debt and verify object absence after delete, including commit-then-raise. |
| T-473-30-04 | Denial of service | owner scans and purge pages | mitigate | Bound each pass and persist independent real continuation tokens, retry debt, and quiescence epochs. |
| T-473-30-05 | Spoofing | aggregate branch result | mitigate | Accept only registered branch IDs and current command generation/version with exact cardinality. |
| T-473-30-SC | Tampering | package installs | mitigate | No package-manager install is planned; retain the standard package-legitimacy gate if scope changes. |
</threat_model>
