# Plan 142-01: Implement Visual Localization And Language Preference UI

## Scope

Wire the existing frontend language switcher to the backend locale preference API, align runtime language options to the backend-supported English/German set, and add browser coverage for persistence through refresh.

## Tasks

1. Align supported runtime languages to `en` and `de`.
2. Remove active French/Italian resource registration and typed locale layout entries from runtime maps.
3. Add locale preference API client and mutation hook.
4. Update the current-user query to apply `/auth/me` locale state to i18next.
5. Update `LanguageSwitcher` variants to call the backend preference mutation for authenticated users.
6. Keep display labels separate from canonical codes.
7. Add Playwright coverage for changing to German, observing the API body, seeing translated UI, and persisting through reload.
8. Run lint, production build, targeted localization e2e, and the mobile regression e2e from Phase 141.
