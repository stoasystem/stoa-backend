# Plan: 138-01 Implement digest and push preference readiness

**Phase:** 138
**Status:** Complete
**Requirement:** NOTIFYDEL-03

## Steps

1. Add a digest preview service that selects visible unread notifications for the authenticated recipient.
2. Support optional category and time-window filters.
3. Produce a stable digest item shape with event, target, title, summary, timestamp, category, and sanitized metadata.
4. Expose the preview through `GET /notifications/digest-preview`.
5. Include explicit no-provider readiness fields for email and push behavior.
6. Add focused tests for digest preference interaction, metadata safety, time-window filtering, and no-provider fallback.

## Acceptance Mapping

- Digest selection by recipient/category/window: `notification_service.digest_preview`.
- Metadata-safe payload shape: `_digest_safe_metadata` and `NotificationDigestItem`.
- Push preference readiness: existing durable `push` preference flag plus preview readiness metadata.
- No-provider fallback: preview response reports `deliveryMode=preview_only`, `emailProviderConfigured=false`, and `pushProviderConfigured=false`.
