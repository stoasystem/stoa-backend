# Phase 252 Plan

## Goal

Define the exact v5.16 release evidence contract from current backend and frontend reality.

## Tasks

1. Map release-critical product surfaces to backend routes, services, and tests.
2. Map the same surfaces to frontend pages/components and focused e2e specs.
3. Reconcile v5.12-v5.15 local completion evidence against current v5.16 requirements.
4. Preserve the v5.14 focused frontend e2e blocker as a release risk until Phase 253 proves it closed.
5. Separate implementation gaps from external provider activation blockers.
6. Promote the resulting matrix into the Phase 253-256 execution contract.

## Success Criteria

- Auth, verification, billing, entitlement, usage/quota, curriculum, teacher help, parent account operations, and admin support are mapped to concrete files and tests.
- Local implementation evidence and live-provider blockers are classified separately.
- Phase 253 has an explicit focused e2e command set.
- Phase 254-256 know which backend and cross-surface evidence must be verified.
