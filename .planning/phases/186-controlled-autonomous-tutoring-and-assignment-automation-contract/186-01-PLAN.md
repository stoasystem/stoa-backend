# Plan 186-01: Define Controlled Autonomous Tutoring And Assignment Automation Contract

## Goal

Create a concrete v5.3 contract for controlled autonomous tutoring and assignment automation before implementation expands.

## Work Items

1. Confirm current v3.7, v4.0, v5.1, and v5.2 foundations for reviewed AI drafts, assignments, sequencing, and analytics.
2. Define backend, frontend, tutor/admin, student/parent, analytics, and release ownership boundaries.
3. Define autonomy levels: off, suggest-only, tutor-approved batch, auto-create reviewed, and future auto-deliver.
4. Define eligible sources: accepted AI practice drafts, published curriculum exercises, and v5.2 recommendation candidates.
5. Define automation policy fields, candidate batch shape, duplicate/stale/refusal rules, and pause behavior.
6. Define assignment creation/delivery states, idempotency keys, result evidence, and role-visible payloads.
7. Define tutor/admin review UX handoff and parent/student explanation contract.
8. Define Phase 187 through Phase 190 implementation targets and verification evidence.

## Verification

- Contract maps every AUTOASSIGN-01 acceptance criterion to implementation targets.
- Unreviewed AI assignment publication and fully autonomous tutoring remain out of scope.
- Follow-up phases have concrete code, docs, and test targets.

- `186-VERIFICATION.md`
