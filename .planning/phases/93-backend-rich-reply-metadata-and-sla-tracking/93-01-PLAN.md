## Objective

Extend the backend teacher question flow so teacher replies can store a safe versioned
rich/formula payload and SLA timing evidence without new infrastructure.

## Work Items

1. Add a backend helper for allowed reply blocks, formula validation, privacy marker refusal, normalized rich payload storage, and plain-text fallback generation.
2. Record `teacher_requested_at` and `queue_visible_at` when a student escalates a question.
3. Record `teacher_taken_over_at`, retain `teacher_started_at`, and store request-to-takeover SLA seconds during takeover.
4. Store `teacher_response`, `teacher_response_text`, `teacher_response_rich`, `teacher_response_format`, `teacher_first_replied_at`, request-to-first-reply seconds, and takeover-to-first-reply seconds during first reply.
5. Refuse teacher replies for missing, unauthorized, or resolved questions.
6. Extend admin stats with aggregate teacher SLA metrics only, with no question content or private markers.
7. Add focused backend tests for plain text compatibility, formula payloads, unsafe marker refusal, authorization/state refusal, and SLA aggregation.
