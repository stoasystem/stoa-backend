# Plan 177-01: Define Rich Editor UI And API Readiness

## Goal

Prepare the admin/tutor rich curriculum editor UI and backend API readiness handoff without changing production content or student/parent published reads.

## Work Items

1. Inventory current backend authoring routes, request/response models, validation, publish, rollback, archive, and analytics behavior.
2. Define the rich editor content model handoff for sections, formulas, media references, exercise blocks, answer keys, hints, explanations, tags, prerequisites, locale metadata, and duration.
3. Define UI layout, states, validation, diff/preview, review queue, and publish/rollback/archive interaction contracts.
4. Identify backend field gaps between current v4.6 MVP authoring payloads and the desired rich editor payload.
5. Preserve published-only student/parent read behavior and answer-key privacy.
6. Record verification evidence and follow-up implementation tasks.
