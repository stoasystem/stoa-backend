# Plan 33-01: Recovery Contract, CDK Readiness, And Dist Guard

## Goal

Define the v1.6 recovery contract and add a fail-fast Lambda dist provenance guard shared by backend direct deploys and infra CDK deploys.

## Scope

- Write Phase 33 context and capture the contract decisions that shape v1.6.
- Add a backend Lambda dist build script that writes and verifies `.stoa-build-manifest.json`.
- Update backend direct deploy workflow to call the shared build script.
- Add an infra CDK synth-time guard that calls backend `--verify-only` before `Code.from_asset`.
- Update infra CDK deploy workflow to call the shared build script.
- Verify tests, lint, full build, manifest verify, and CDK synth.

## Implementation Tasks

1. Add backend `scripts/build_lambda_dist.py`.
2. Add backend tests for manifest verification, stale source detection, missing handler detection, and timestamp-stable CDK asset hashing.
3. Replace duplicated backend workflow build commands with the shared script.
4. Add infra `stacks/lambda_dist_guard.py`.
5. Update infra `stacks/api_stack.py` to verify `dist` before asset loading and use manifest `cdk_asset_hash`.
6. Replace duplicated infra workflow build commands with the shared backend script.
7. Record verification evidence and update planning metadata.

## Risks

- Full Lambda build needs PyPI/network access for manylinux arm64 wheels.
- Local manifest can record `source_git_dirty=true` while work is uncommitted; CI should record a clean checkout.
- `ALLOW_STALE_LAMBDA_DIST=1` is an emergency override and must not be set in normal CI/CD.
