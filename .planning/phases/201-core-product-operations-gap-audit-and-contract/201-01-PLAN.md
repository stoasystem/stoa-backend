# Plan 201-01: Audit And Define Core Product Operations Completion Contract

## Goal

Create a code-grounded v5.6 audit and development plan for paid entitlements, usage records, login verification code policy, email verification, customer billing state, and admin support visibility.

## Work Items

1. Audit current auth/register/login/refresh/logout/forgot/reset behavior and identify real email verification and login-code gaps.
2. Audit current subscription/billing/manual plan behavior and identify missing paid entitlement state transitions.
3. Audit current quota and usage recording behavior, including daily question usage and any AI/OCR/teacher-help usage.
4. Define effective entitlement inputs: subscription tier, billing provider state, manual admin override, rollout controls, cancellation/expiry, and pending payment.
5. Define usage ledger event model: actor, subject, product area, action, entitlement source, period, idempotency key, and admin query dimensions.
6. Define email verification and login-code lifecycle: requested, sent, verified, expired, failed, resent, wrong code, already verified, and account effects.
7. Define customer/admin API visibility and Phase 202 through Phase 205 implementation targets.
8. Record the current reality audit and next development task queue.

## Verification

- Current reality audit maps every COREOPS-01 acceptance criterion to code evidence and implementation targets.
- Native app work is explicitly deferred until paid/auth/usage correctness is addressed.
- Follow-up phases have concrete backend, API, admin/customer visibility, and focused-test targets.
