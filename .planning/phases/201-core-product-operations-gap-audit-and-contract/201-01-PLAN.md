# Plan 201-01: Audit And Define Final-Polish Milestone Contract

## Goal

Create a code-grounded final-polish audit and milestone plan for paid entitlements, usage records, login verification code policy, email verification, customer billing state, and admin support visibility.

## Work Items

1. Audit current auth/register/login/refresh/logout/forgot/reset behavior and identify real email verification and login-code gaps.
2. Audit current subscription/billing/manual plan behavior and identify missing paid entitlement state transitions.
3. Audit current quota and usage recording behavior, including daily question usage and any AI/OCR/teacher-help usage.
4. Define effective entitlement inputs: subscription tier, billing provider state, manual admin override, rollout controls, cancellation/expiry, and pending payment.
5. Define usage ledger event model: actor, subject, product area, action, entitlement source, period, idempotency key, and admin query dimensions.
6. Define email verification and login-code lifecycle: requested, sent, verified, expired, failed, resent, wrong code, already verified, and account effects.
7. Define customer/admin API visibility and promote the remaining work into complete v5.6-v5.9 feature milestones.
8. Define v5.6 internal phases 202 through 206 for effective entitlement delivery.
9. Record the current reality audit and next development milestone queue.

## Verification

- Current reality audit maps every core-operations criterion to code evidence and implementation targets.
- Native app work is explicitly deferred until entitlement, usage, verification, and operations visibility correctness is addressed.
- Follow-up milestones have concrete backend, API, admin/customer visibility, and focused-test targets.
