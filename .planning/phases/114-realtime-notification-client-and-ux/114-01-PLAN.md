Status: in_progress

Add frontend realtime notification client behavior across role shells while preserving the existing notification center and polling fallback.

- Client handles reconnect, heartbeat, offline/unavailable state, and fallback to existing notification list polling.
- Student, parent, tutor, and admin shells show realtime notification count/list updates through the shared shell.
- Tutor workflows can receive teacher-session notification events without page refresh.
- Targeted browser verification proves realtime and fallback UX with local or safe test fixtures.
