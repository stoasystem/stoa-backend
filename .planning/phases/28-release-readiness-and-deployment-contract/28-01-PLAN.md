## Threat Model

<threat_model>
  <threat id="TM-28-01" severity="high">
    <risk>Production recovery mutations could run against a real customer report before a safe smoke target is documented.</risk>
    <mitigation>`28-RELEASE-READINESS.md` must include a blocking mutation safety gate requiring no-customer-PII confirmation and explicit target identifiers before Phase 31 mutation smoke.</mitigation>
  </threat>
  <threat id="TM-28-02" severity="medium">
    <risk>CDK diff output could be misread by treating unexpected IAM or bucket drift as ordinary Lambda code asset drift.</risk>
    <mitigation>The CDK diff classification policy must separate expected Lambda `Code.S3Key` changes from unexpected IAM, bucket, API route, DynamoDB, or policy drift and block live mutation smoke for unexpected drift.</mitigation>
  </threat>
  <threat id="TM-28-03" severity="medium">
    <risk>Rollback could be improvised after a failed deployment instead of being known before production verification begins.</risk>
    <mitigation>The release readiness contract must identify backend Lambda, frontend asset, and infra rollback entry points before Phase 29 or Phase 30 starts.</mitigation>
  </threat>
</threat_model>
