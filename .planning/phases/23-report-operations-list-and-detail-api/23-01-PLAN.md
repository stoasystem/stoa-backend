# Plan 23-01: Report Operations List and Detail API

## Objective

Deliver the backend admin report operations list/detail API for Phase 23, including pagination/filter access, generation metadata, action eligibility, privacy boundaries, and focused tests.

## Tasks

1. Extend `report_repo.py`
   - Add opaque DynamoDB pagination token encode/decode helpers.
   - Add `list_reports_for_admin(...)`.
   - Use `GSI-ParentId` for parent-specific queries where possible.
   - Use bounded scan for cross-parent pilot admin listing.

2. Extend `admin.py`
   - Add list request/response models.
   - Add `GET /admin/reports/ops`.
   - Expand detail response with generation metadata.
   - Add action eligibility with disabled reasons.
   - Preserve admin-only and metadata-only response boundaries.

3. Add backend tests
   - Admin-only list access.
   - List response with filters, pagination token, and metadata fields.
   - Detail response includes generation metadata and eligibility.
   - Privacy assertions continue to reject raw content/public URL leakage.
   - Repository tests cover scan/query call shape and token round trip.

4. Verification and docs
   - Run focused pytest for admin report ops and report repository behavior.
   - Record OPS-03 access-pattern evidence.
   - Write `23-01-SUMMARY.md` and `23-VERIFICATION.md`.
   - Update roadmap/state progress for Phase 23.

## Acceptance Criteria

- OPS-01 through OPS-05 are covered.
- `GET /admin/reports/ops` is admin-only and returns bounded, paginated metadata rows.
- Existing detail endpoint returns generation metadata and action eligibility.
- No response exposes raw report content or S3 URLs.
- Focused tests pass.
