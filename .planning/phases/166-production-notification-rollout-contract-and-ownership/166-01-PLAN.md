# Plan 166-01: Define Production Notification Rollout Contract And Ownership

## Goal

Create a concrete v4.9 rollout contract for production WebSocket delivery, provider-backed email/push notifications, frontend/native UX handoff, native push token registration, rollout gates, observability, and live smoke evidence.

## Work Items

1. Inspect current v3.6 and v4.2 notification implementation and release evidence.
2. Define backend, frontend, native, infrastructure, and provider ownership boundaries.
3. Define live WebSocket/API Gateway route expectations, auth/subscription model, runtime configuration, fallback behavior, and deployment prerequisites.
4. Define email digest provider mode, scheduling/manual trigger readiness, templates, recipient selection, preference gating, and send/refusal/failure evidence.
5. Define push provider mode, native token registration, token lifecycle, preference gating, and send/refusal/failure evidence.
6. Define in-scope notification event categories for live rollout and which categories remain backend-only.
7. Define admin/operator observability, rollout states, rollback behavior, and live smoke boundaries.
8. Write Phase 167 through 170 implementation handoff.

## Verification

- Contract maps every PRODNOTIF-01 acceptance criterion to implementation targets.
- External infrastructure/provider/native prerequisites are explicit and actionable.
- No real provider-backed user notification send is performed.
- Follow-up phases have concrete code and test targets.
