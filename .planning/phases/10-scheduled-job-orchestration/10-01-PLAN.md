# Phase 10 Plan 01: Scheduled Weekly Report Job

## Goal

Implement the scheduled weekly report Lambda handler so it processes eligible linked parent/student pairs idempotently and returns structured job counts.

## Tasks

### 1. Implement target week calculation

File:

- `src/stoa/jobs/weekly_reports.py`

Work:

- Parse explicit `week_start` / `weekStart` when present.
- Otherwise parse EventBridge `time` or current time.
- Calculate previous Zurich calendar week start.

Acceptance:

- Scheduled events default to the previous Zurich week.
- Tests can inject `now` for deterministic behavior.

### 2. Implement linked pair discovery

File:

- `src/stoa/jobs/weekly_reports.py`

Work:

- Scan user profiles for `role = student` and present `parent_id`.
- Page through scan results.
- Return parent/student pair dictionaries.

Acceptance:

- Eligible linked pairs are discovered without manually supplied IDs.

### 3. Implement idempotent orchestration

File:

- `src/stoa/jobs/weekly_reports.py`

Work:

- Add a testable `run_weekly_report_job(event, now=None)` function.
- For each pair, skip existing report records for `(parent_id, student_id, week_start)`.
- For new pairs, call aggregation, generation, and storage/email services in order.
- Count email_sent vs generated vs failed from returned statuses.

Acceptance:

- Duplicate runs skip existing generated or email-sent reports.
- Per-pair failures do not abort the whole job.

### 4. Add focused tests

File:

- `tests/test_weekly_reports_job.py`

Work:

- Test previous Zurich week calculation and explicit event override.
- Test pair discovery pages through scans.
- Test duplicate reports are skipped before generation.
- Test successful pair processing returns attempted/generated/email counts.
- Test per-pair failure increments failed and continues.

Acceptance:

- Tests pass without AWS clients by monkeypatching table/repositories/services.

### 5. Verify

Commands:

- `uv run --extra dev pytest tests/test_weekly_reports_job.py -q`
- `uv run --extra dev ruff check src/stoa/jobs/weekly_reports.py tests/test_weekly_reports_job.py`

## Verification Criteria

- `JOB-01` is satisfied when handler accepts scheduled events and defaults to previous Zurich week.
- `JOB-02` is satisfied when linked parent/student pairs are discovered from profiles.
- `JOB-03` is satisfied when existing reports are checked by `(parent_id, student_id, week_start)`.
- `JOB-04` is satisfied when duplicate runs skip existing generated or email-sent reports.
- `JOB-05` is satisfied when structured counts are returned.

## Risks

- Scan-based pair discovery is MVP tech debt, consistent with previous phases.
- Live EventBridge invocation is not exercised locally; tests cover event shape and handler response.
