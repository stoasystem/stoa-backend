# Plan 26-01: Admin Report Operations UI

## Objective

Add an admin report operations page that uses the real backend report operations APIs for triage, detail inspection, generation retry, single resend, and selected bulk resend.

## Tasks

1. Add frontend API contracts
   - Add report operation list/detail types.
   - Add list/detail API clients.
   - Add retry, resend, and bulk resend API clients.

2. Add React Query hooks
   - Add list/detail queries.
   - Add mutation hooks for retry, resend, and bulk resend.
   - Invalidate report operations queries after successful mutations.

3. Build admin UI page
   - Add filters for status, week start, parent ID, and student ID.
   - Show loading, empty, and error states.
   - Show paginated report rows with status badges and action eligibility.
   - Support row selection and detail inspection in-page.
   - Support eligible single retry/resend actions.
   - Support selected bulk resend for eligible `email_failed` reports.
   - Render per-action and per-item results.

4. Wire navigation and route
   - Add `/admin/report-operations`.
   - Add admin nav item.

5. Verification
   - Run frontend build/lint as feasible.
   - Verify no silent demo fallback is used.
   - Write summary, review, and verification artifacts.
   - Update roadmap, requirements, and state for Phase 26 completion.

## Acceptance Criteria

- UI-01 through UI-05 are covered.
- The page uses real admin report operations APIs.
- The page does not expose raw report HTML/JSON, private S3 keys, public URLs, presigned URLs, or direct S3 fetch paths.
- Action controls follow backend eligibility metadata.
