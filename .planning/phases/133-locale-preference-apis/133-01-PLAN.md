# Phase 133 Plan: Locale Preference APIs

**Phase:** 133
**Requirement:** I18N-01
**Status:** Planned
**Created:** 2026-06-11

## Objective

Implement durable locale preference APIs and shared normalization/fallback behavior while preserving compatibility with existing auth/profile clients.

## Tasks

1. Add shared locale normalization and effective-locale helper.
2. Add user repository helper for durable locale updates.
3. Extend `/auth/me` response with `preferredLocale` and `effectiveLocale`.
4. Add authenticated locale update route.
5. Add focused tests for supported, missing, regional, unsupported, and malformed locale behavior.
6. Run focused auth/locale tests and ruff on changed files.

## Acceptance Criteria Mapping

| Criterion | Plan coverage |
|-----------|---------------|
| Authenticated profile/preference responses expose effective locale. | `/auth/me` response model and tests. |
| Supported locale updates persist durably. | `PATCH /auth/me/preferences/locale` and repository helper. |
| Missing/malformed/unsupported/supported behavior is deterministic and tested. | `locale_service` and tests. |
| Existing authorization and clients remain compatible. | Existing `preferredLanguage` preserved; auth lifecycle tests rerun. |
