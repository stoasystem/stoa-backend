# Phase 65 Plan: v2.3 Release Gate And Milestone Audit

## Objective

Close v2.3 with release evidence proving the release evidence automation and fixture lifecycle workflow is deployed, admin-only where applicable, privacy-safe, and production-verified without customer-impacting mutation.

## Tasks

1. Record backend and frontend commit SHAs, deploy workflow run IDs, job IDs, timestamps, and deployment status.
2. Record Lambda build manifest, deployed Lambda runtime metadata, and CDK diff/deploy classification.
3. Record local quality gates for backend release evidence tooling and frontend release evidence UI.
4. Run production read-only API smoke with secret-backed admin auth against release evidence validation and fixture status endpoints.
5. Run production read-only browser smoke for `/admin/report-operations` with a non-mutation request guard and verify release evidence UI markers.
6. Run safe-fixture mutation refusal check; run fixture mutation smoke only if explicitly approved and record cleanup/restore evidence.
7. Audit all v2.3 requirements and record residual risks, blockers, deferred requirements, and closeout decision.

## Verification Commands

Backend:

- `.venv/bin/python -m ruff check src/stoa/services/release_evidence_service.py src/stoa/routers/admin.py scripts/release_evidence.py tests/test_release_evidence.py`
- `.venv/bin/python -m pytest tests/test_release_evidence.py -q`
- `.venv/bin/python scripts/release_evidence.py validate --input <redacted-release-evidence.json>`
- `.venv/bin/python scripts/release_evidence.py fixture-status --input <redacted-fixture-status.json>`
- `.venv/bin/python scripts/release_evidence.py check-mutation --input <redacted-mutation-request.json>`

Frontend:

- `npm run lint -- src/services/admin/adminApi.ts src/hooks/admin/useAdminReportOperations.ts src/pages/admin/ReportOperationsPage.tsx tests/e2e/admin-report-operations.spec.ts`
- `npm run build`
- `npx playwright test tests/e2e/admin-report-operations.spec.ts`

Production:

- `uv run cdk diff --profile stoa-prod-admin`
- Production read-only API smoke for `/admin/reports/release-evidence/validate` and `/admin/reports/release-evidence/fixture-status`.
- Production read-only browser smoke for `https://app.stoaedu.ch/admin/report-operations`.
- Safe-fixture mutation refusal check.

## Exit Criteria

Phase 65 can be marked complete only when:

- `65-RELEASE-GATE.md` records deploy/build/CDK/runtime/quality evidence.
- `65-LIVE-VERIFICATION.md` records production read-only API/browser smoke with request IDs and clean privacy denylist results.
- Safe-fixture mutation is either explicitly skipped with a reason or performed only through the approved fixture with cleanup/restore evidence.
- `65-MILESTONE-AUDIT.md` marks each v2.3 requirement complete or records a blocker.
