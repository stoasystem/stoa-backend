# Plan 164-01: Implement Support SLA Analytics And Controlled CRM Messaging

<objective>
Add metadata-only SLA analytics and controlled support/customer messaging evidence for support handoff deliveries.
</objective>

<tasks>
<task id="1" type="auto">
  <name>Add SLA analytics service</name>
  <files>
    - src/stoa/services/support_sla_service.py
    - src/stoa/db/repositories/report_repo.py
  </files>
  <action>
    Build aggregate SLA metrics from support handoff delivery summaries, including status counts, overdue deliveries, provider failure rate, retry backlog, conflict count, and message outcome counts.
  </action>
  <acceptance_criteria>
    - Analytics cover queued, delivered, acknowledged, first response/in progress, resolved, failed, and reopened states.
    - Missing or malformed timestamps do not crash analytics.
    - Responses include only metadata-safe delivery references.
  </acceptance_criteria>
</task>

<task id="2" type="auto">
  <name>Add controlled message evidence</name>
  <files>
    - src/stoa/config.py
    - src/stoa/db/repositories/report_repo.py
    - src/stoa/services/support_sla_service.py
    - src/stoa/routers/admin.py
  </files>
  <action>
    Add approved-template support messaging that persists send/refusal/failure evidence correlated to a support delivery without sending raw payloads or freeform customer text.
  </action>
  <acceptance_criteria>
    - Supported templates: support receipt, status update, resolution, escalation.
    - Messaging requires destination approval and template approval.
    - Evidence includes delivery ID, package ID, template, destination, outcome, correlation ID, actor, and redacted refusal/failure reasons.
  </acceptance_criteria>
</task>

<task id="3" type="auto">
  <name>Add admin endpoints and tests</name>
  <files>
    - src/stoa/routers/admin.py
    - tests/test_admin_report_ops.py
  </files>
  <action>
    Expose SLA analytics and controlled messaging endpoints, then cover success, overdue classification, provider failure analytics, approval refusal, opt-out refusal, provider failure, and admin-only access.
  </action>
  <acceptance_criteria>
    - Focused support handoff tests pass.
    - Ruff passes for touched files.
    - Private artifact markers remain absent from responses and persisted evidence.
  </acceptance_criteria>
</task>
</tasks>
