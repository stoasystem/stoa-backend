# Plan 167-01: Implement Live WebSocket Deployment Readiness And Status

## Goal

Make backend WebSocket delivery readiness observable for production rollout while preserving durable notification fallback.

## Work Items

1. Add runtime settings for live WebSocket route readiness, deployment state, smoke state, and stale cleanup enablement.
2. Add a WebSocket readiness helper that reports endpoint mode, endpoint host, route/deploy/smoke flags, active/stale connection counts, cleanup state, and configuration blockers.
3. Extend admin notification delivery status with readiness mode, recent redacted delivery attempt summaries, aggregate fanout result counts, and the existing preference/category status.
4. Keep fanout persistence/fallback behavior unchanged.
5. Add focused tests for local-only and live-ready status, redacted attempt evidence, stale cleanup visibility, and fallback-compatible fanout.
