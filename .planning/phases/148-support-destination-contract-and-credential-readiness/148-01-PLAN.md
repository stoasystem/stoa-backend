## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| admin request -> support contract | Destination mode, operator note, and evidence-selection inputs must not broaden allowed modes or leak secrets. |
| contract artifact -> future provider adapter | Later delivery code will rely on this document for approved fields, readiness states, and refusal behavior. |
| support package/audit metadata -> operator readers | Metadata-only evidence and credential readiness must remain redacted for humans, logs, and stored audit data. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-148-01 | Tampering | destination mode contract | mitigate | Require an explicit mode allowlist, preserve unknown-mode and `external_write` refusal before evidence reads, and state that Phase 148 adds no third-party live provider write path. |
| T-148-02 | Information Disclosure | credential readiness and contract artifact | mitigate | Permit only secret references, env-var names, presence flags, owners, and approval gates; forbid token values, authorization headers, cookies, and raw secrets in docs or responses. |
| T-148-03 | Information Disclosure | support payload and attachments | mitigate | Allow only redacted summary fields, reference IDs, safe tags, approved custom fields, and payload digests; keep attachments disabled by default and forbid raw report HTML/JSON, S3 keys, and presigned URLs. |
| T-148-04 | Repudiation | future delivery lifecycle contract | mitigate | Separate package validation status from delivery lifecycle state and require correlation ID, idempotency key, redacted refusal/failure reasons, provider object references, and retry classification in the contract. |
| T-148-SC | Tampering | package installs | accept | No package-manager installs or provider SDK selection occur in this docs/contracts-only phase. |
</threat_model>
