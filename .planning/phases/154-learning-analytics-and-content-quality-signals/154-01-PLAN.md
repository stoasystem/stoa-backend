# Phase 154 Plan

**Phase:** 154 - Learning Analytics And Content Quality Signals
**Plan:** 154-01
**Status:** Complete
**Created:** 2026-06-12

<objective>
Add bounded curriculum analytics signals and aggregate content-quality views without adding a warehouse or exposing student-sensitive details.
</objective>

<tasks>

1. Add same-table analytics repository helpers.
   - Files: `src/stoa/db/repositories/curriculum_analytics_repo.py`
   - Acceptance: repository can write signal rows, increment bounded metric rows, and list metrics with content/subject/topic filters.

2. Add analytics service wrappers.
   - Files: `src/stoa/services/curriculum_analytics_service.py`
   - Acceptance: service records practice, wrong-answer, completion, assignment, publish, archive, and aggregate quality responses with privacy flags.

3. Wire analytics into existing flow points.
   - Files: `src/stoa/routers/practice.py`, `src/stoa/services/adaptive_learning_service.py`, `src/stoa/services/curriculum_ops_service.py`
   - Acceptance: practice attempts/completions, assignment completions/skips, publish, rollback-publish, and archive emit aggregate-safe signals.

4. Add operator content-quality endpoint.
   - Files: `src/stoa/routers/admin.py`
   - Acceptance: authorized operators can fetch aggregate quality metrics without raw student answers, answer keys, or student identifiers.

5. Add tests and compatibility verification.
   - Files: `tests/test_curriculum_analytics.py`
   - Acceptance: tests cover signal recording and privacy boundary, while existing curriculum/adaptive/admin tests remain green.

</tasks>
