# Plan 38-01: Credential Ops Contract and Export Design

**Phase:** 38 - Credential Ops Contract And Export Design
**Milestone:** v1.7 Recovery Evidence Export & Admin Credential Operations
**Status:** Planned
**Created:** 2026-06-05

## Objective

Prepare v1.7 implementation by turning the long-lived production admin credential into a documented operational procedure and defining a metadata-only export contract for recovery evidence. This plan is documentation/design only and must not mutate production reports.

## Context

Phase 36 created the long-lived production admin credential path `stoa/production/admin/stoaedu.ad@gmail.com` and used it for read-only production browser smoke. Phase 37 documented the v1.6 report recovery runbook and release gate, but left credential ownership/rotation and metadata-only export as follow-up work.

Existing backend surfaces already expose admin-only metadata for:

- Recovery job list/detail: `/admin/reports/recovery-jobs`.
- Recovery job results: `/admin/reports/recovery-jobs/{job_id}/results`.
- Report audit timeline: `/admin/reports/{parent_id}/{student_id}/{week_start}/audit`.
- Recovery job audit timeline: `/admin/reports/recovery-jobs/{job_id}/audit`.

Existing repository access patterns in `src/stoa/db/repositories/report_repo.py` support recovery job summary scans, job target queries, report audit queries, and job audit queries. Phase 38 must decide whether Phase 39 can compose these existing patterns safely or needs a narrower repository helper.

## Tasks

### Task 1: Credential Lifecycle Runbook

Create a Phase 38 credential operations document or update the v1.6 runbook with:

- Production admin credential path: `stoa/production/admin/stoaedu.ad@gmail.com`.
- Required AWS profile/region for operator verification: `stoa-prod-admin`, `eu-central-2`.
- Named owner field, even if the owner is marked `TBD by operations`.
- Rotation cadence and exact rotation steps.
- Cognito admins group membership verification steps.
- Emergency disable/revoke steps.
- Access review checklist.
- Evidence fields to capture: timestamp, redacted username, Cognito user pool ID, group name, AWS account/region, command/request IDs where available.

Do not include passwords, temporary passwords, access tokens, ID tokens, refresh tokens, browser cookies, or session storage dumps.

### Task 2: Export Contract

Create a Phase 38 export contract document covering:

- Request modes:
  - `job_id` export for one recovery job.
  - Bounded recent jobs export with `limit` and optional pagination token.
  - Optional time/status filter only if existing access patterns can support it safely.
- Response sections:
  - Export metadata.
  - Job summaries.
  - Target result summaries.
  - Job audit summaries.
  - Optional report audit summaries only when a specific report identity is supplied.
- Allowlisted fields:
  - IDs, statuses, timestamps, counters, actor/operator identifiers, reasons after redaction, filters after redaction, result classes, redacted details, and request IDs.
- Denied fields:
  - `weekly-reports/`.
  - `json_s3_key`, `html_s3_key`, `s3_key`, and any key ending in `_s3_key`.
  - Presigned URLs and public S3 URLs.
  - Raw report JSON or HTML.
  - Auth/session tokens.
  - Full DynamoDB raw items.
- Bounds:
  - Conservative default and maximum limits.
  - Pagination behavior.
  - Failure behavior for unbounded requests.
- Read-only guarantee:
  - No job creation.
  - No report status updates.
  - No resend/retry invocation.
  - No S3 read/write.
