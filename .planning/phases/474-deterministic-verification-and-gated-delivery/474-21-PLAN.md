## Trust Boundaries

| Boundary | Description |
|---|---|
| pull request/dispatch inputs -> workflow | Attacker-controlled refs and fields must not become commands or privileged checkout targets. |
| Web workflow -> common gate | A local workflow could duplicate or weaken backend/Web verification. |
| verified Web artifact -> delivery handoff | Bundle or manifest substitution could deploy unverified bytes. |

## STRIDE Threat Register

| Threat ID | Severity | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-474-21-01 | Critical | Elevation of privilege | workflow events/permissions | mitigate/block | Use least permissions, no privileged pull-request-target path, and no secrets on untrusted events. |
| T-474-21-02 | Critical | Tampering | candidate/action refs | mitigate/block | Require full candidate SHAs and full action commit SHAs; reject branch and mutable tag identities. |
| T-474-21-03 | Critical | Tampering | Web artifact handoff | mitigate/block | Pass only manifest-bound digest/receipt identities; coordinator rehashes before effects. |
| T-474-21-04 | High | Repudiation | workflow provenance | mitigate/block | Emit workflow/run/attempt/commit/manifest IDs into the canonical receipt. |
| T-474-21-05 | High | Denial of service | duplicate candidate runs | mitigate | Candidate-keyed concurrency cancels superseded unprivileged verification only. |
| T-474-21-SC | High | Tampering | action supply chain | mitigate/block | Freeze every third-party action to a reviewed full commit SHA; no package install is introduced here. |
</threat_model>
