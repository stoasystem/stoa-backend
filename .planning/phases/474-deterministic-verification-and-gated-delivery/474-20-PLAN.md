## Trust Boundaries

| Boundary | Description |
|---|---|
| manifest/artifacts -> AWS effects | Source or bytes may be substituted after verification. |
| transaction -> two independent pointers | Concurrency/partial failure can split backend and Web release identity. |
| authorization/rollback input -> production | A confused deputy could promote or roll back an unverified set. |

| Threat ID | Severity | Category | Component | Disposition | Mitigation Plan |

| T-474-20-02 | Critical | Tampering | pointer race | mitigate/block | Durable prior/target transaction, conditional versions, read-after-write verification. |
| T-474-20-03 | Critical | Elevation of privilege | production/rollback auth | mitigate/block | Environment-scoped role plus separate exact authorization; verified-manifest rollback only. |
