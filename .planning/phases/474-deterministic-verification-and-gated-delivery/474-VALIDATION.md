# Phase 474 — Validation Strategy

> Per-phase validation contract for feedback sampling during execution. Every final-plan task has an automated command; Plan 14 has a conditional exact-residual owner decision, Plan 24 has a protected-environment owner checkpoint, and Plan 83 fails closed for owner resolution of any pre-existing sibling-repository dirt.

## Test Infrastructure

| Property | Value |
|---|---|
| **Framework** | pytest + Ruff + mypy; Node test + ESLint + TypeScript + Playwright; CDK assertions |
| **Config file** | `pyproject.toml`, frontend `package.json`/Playwright configs, infra `pyproject.toml` |
| **Quick run command** | Use the exact task-row command below after each task commit |
| **Full suite command** | `python scripts/release_gate.py verify --backend-ref <sha> --frontend-ref <sha> --infra-ref <sha> --receipt-root <path>` |
| **Estimated runtime** | Focused checks target under 60 seconds; formal two-clock/Web/staging checks retain progress receipts |

## Sampling Rate

- **After every task commit:** Run that task's exact automated row command.
- **After every plan wave:** Run all task commands in the completed wave and validate their canonical receipts.
- **Before `$gsd-verify-work`:** Run the authoritative full cross-repository gate from clean exact commits.
- **Max feedback latency:** 60 seconds for focused checks; long formal/staging checks emit retained progress and never use watch mode.
