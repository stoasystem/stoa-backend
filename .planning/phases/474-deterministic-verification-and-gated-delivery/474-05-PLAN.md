<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Git repositories -> manifest | Refs, dirty trees, submodule/tree bytes, and lockfiles are untrusted. |
| build tree -> artifact | ZIP metadata/order or frontend config can change bytes after verification. |
| manifest -> deploy | A digest or gate substitution could promote unverified content. |

## STRIDE Threat Register

| Threat ID | Severity | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-474-05-01 | Critical | Tampering | artifact bytes | mitigate/block | Hash final bytes, normalize ZIP metadata, and revalidate before promotion. |
| T-474-05-02 | High | Spoofing | cross-repo identity | mitigate/block | Exact detached commits/tree/lock identities; dirty or mutable refs rejected. |
| T-474-05-03 | High | Repudiation | verification binding | mitigate/block | Require exact gate receipt IDs/digests and complete closed result set. |
| T-474-05-04 | High | Information disclosure | manifest | mitigate/block | IDs, hashes, counts, versions, and status only; no secrets or browser/private payloads. |
| T-474-05-SC | High | Tampering | package installs | mitigate/block | No new package; build uses frozen lock and verified production export only. |
</threat_model>
