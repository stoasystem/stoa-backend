## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CDK source -> planning artifacts | Executor translates external infrastructure code into milestone assumptions. False evidence here would mislead later implementation. |
| Cognito JWT claims -> local DynamoDB user profiles | Parent identity crosses from external IdP claims into local ownership checks. |
| Parent ID/child ID route inputs -> future backend reads | Untrusted client-controlled IDs must not drive child/report reads before ownership is verified. |
| Backend settings -> AWS resources | Environment variables connect runtime code to AWS resources and permissions. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Spoofing | Parent identity contract | mitigate | Require canonical parent resolution from authenticated claims plus local profile lookup; do not allow future `/parents/me/...` ownership checks to trust a client-supplied parent ID. |
| T-01-02 | Tampering | Infrastructure audit artifact | mitigate | Require file path and line-level CDK/backend evidence for every resource/env assertion; mark unknowns as missing instead of filling assumptions. |
| T-01-03 | Information Disclosure | Future child summary/history/report reads | mitigate | Document invariant that ownership is verified before reading or returning child-specific data. |
| T-01-04 | Denial of Service | Parent-child lookup access pattern | mitigate | Classify child lookup as index-backed, scan-based MVP with explicit pagination/scale risk, or blocked pending CDK-backed GSI. |
| T-01-05 | Repudiation | Phase 1 decisions | mitigate | Record final infrastructure and identity decisions in versioned phase artifacts with requirement IDs. |
| T-01-SC | Tampering | Package installs | accept | This plan performs no package-manager installs and adds no dependencies. |
