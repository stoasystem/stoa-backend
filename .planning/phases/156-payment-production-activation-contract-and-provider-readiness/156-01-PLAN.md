# Plan 156-01: Define Payment Production Activation And Provider Readiness Contract

## Goal

Create a concrete v4.7 activation contract for live Stripe/TWINT readiness, webhook registration, direct refunds, finance acceptance, and rollout controls.

## Work Items

1. Inspect current v4.4 payment implementation and release evidence.
2. Define live credential ownership, injection path, provider mode validation, and redacted readiness output.
3. Define Standard/Premium live price mapping, CHF expectations, TWINT eligibility checks, and provider capability evidence.
4. Define webhook endpoint registration requirements, required events, secret ownership, and last-observed event evidence.
5. Define direct refund execution controls, idempotency, operator reason, and audit requirements.
6. Define finance acceptance requirements for invoices, refunds, tax/accounting handoff, dunning, and reconciliation.
7. Define rollout control model for checkout activation, refund activation, canary state, and rollback.
8. Write Phase 157 through 160 implementation handoff.

## Verification

- Contract maps every PAYACT-01 acceptance criterion to implementation targets.
- External blockers are explicit and actionable.
- No real customer charge is performed.
- Follow-up phases have concrete code and test targets.
