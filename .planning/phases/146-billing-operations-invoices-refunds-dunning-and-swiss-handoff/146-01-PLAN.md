# Plan 146-01: Implement Billing Operations, Swiss Handoff, And TWINT Lifecycle Readiness

## Goal

Add first-pass billing operations readiness for invoices, receipts, refunds, dunning, Swiss accounting handoff, and TWINT lifecycle metadata.

## Work Items

1. Inspect Phase 145 billing projections and provider lookup/event data.
2. Define or implement billing operations models for invoices, receipts, refunds, dunning states, and accounting export rows.
3. Surface provider-hosted invoice/receipt metadata to parents and admins where available.
4. Add refund eligibility and operator handoff state without enabling broad uncontrolled refund mutation.
5. Project `past_due`, `payment_failed`, retry/recovery, cancellation, and escalation states into parent/admin billing views.
6. Add Swiss accounting handoff fields for tier, currency, gross/net/tax metadata where available, provider IDs, invoice IDs, payment method context, and reconciliation references.
7. Include TWINT-originated payment context in the same billing model when Stripe provides it.
8. Add focused tests for invoice metadata, refund eligibility, dunning visibility, accounting export shape, and TWINT lifecycle data.

## Verification

- `pytest tests/test_subscription_operations.py`
- Additional targeted tests if new billing operations tests are split out.
- Focused static checks for touched billing files.

## Non-Goals

- Full accounting integration.
- Broad automated dunning campaigns.
- Direct support-ticket integration.
- Native app payment flow changes.

## Output

- Billing operations implementation or readiness contracts.
- Updated tests.
- `146-BILLING-OPERATIONS-SWISS-HANDOFF-SPEC.md` kept aligned with implementation.
- `146-VERIFICATION.md` with evidence and unresolved provider blockers.
