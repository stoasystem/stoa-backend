# Plan 472-06: Student, question, and conversation route migration

<objective>
Close the direct SEC-002 paths first by migrating every student/question/conversation identifier route to the central policy and canonical Actor identity, with positive and denial controls.
</objective>

## Artifacts this phase produces

- Authorization specs/resolvers for all routes registered from `students.py`, `questions.py`, and `conversations.py`.
- Canonical self-service identity derivation; explicit legacy student IDs must equal Actor or pass related-actor policy.
- Route-family matrix rows for profile/summary/questions, question create/read/teacher-request/feedback, conversation list/create/read/message/stream.

<threat_model>
Assets: student profile/learning history, question content/result, conversation messages and resource existence. Trust boundaries: path/query/body ID to stored owner; authorized object to handler. Abuse cases: another student substitutes `student_id`, unrelated parent/teacher reads known question, conversation ID resolves across students, create body attributes ownership to victim, real-vs-random ID oracle. Mitigations: derive self ID from Actor, owner-resolving policy dependencies, handler consumes authorized object, existence-hiding 404, current relationship/task facts, positive controls. Validation: T-472-06 and T-472-07; direct SEC-002 reproductions are mandatory.
</threat_model>
