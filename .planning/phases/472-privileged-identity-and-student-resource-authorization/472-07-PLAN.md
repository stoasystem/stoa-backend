# Plan 472-07: Practice, adaptive, and parent route migration

<objective>
Apply the central policy to all practice/adaptive student facts and parent child/report routes, eliminating legacy parent/email/one-sided authorization while preserving legitimate self and active-binding flows.
</objective>

## Artifacts this phase produces

- Actor-derived policy specs for every `practice.py` and `adaptive.py` route, including body/query/indirect assignment IDs.
- Bidirectional active parent-binding policy on every `parents.py` child/report path.
- Matrix coverage for practice/progress/mistakes/hints/help, adaptive memory/recommendations/assignments/automation, and parent child/usage/history/report.

<threat_model>
Assets: practice progress/mistakes, adaptive memory/assignments, child learning/usage/report data. Trust boundaries: implicit self routes and explicit/body IDs to canonical student; parent binding rows to access decision. Abuse cases: body/query student ID bypass, assignment ID leaks owner, legacy `parent_id`/email grants access, one-sided/pending binding authorizes, admin/teacher role broadens. Mitigations: Actor self derivation, indirect resource owner resolvers, matching active forward/reverse binding and active accounts, exact action/purpose, hidden 404, positive controls, fresh reads. Validation: T-472-06, T-472-07.
</threat_model>
