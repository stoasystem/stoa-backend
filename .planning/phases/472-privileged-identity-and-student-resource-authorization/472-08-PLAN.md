# Plan 472-08: Teacher task, assistance, and AI-tool route migration

<objective>
Migrate the complete canonical `/teachers` surface to active teacher identity plus exact task/assignment/capability scope; queue/role membership alone must never authorize student content.
</objective>

## Artifacts this phase produces

- Policy specs for teacher availability/stats, queue/dispatch, question takeover/reply/resolve, assistance, help requests/notes, and AI draft lifecycle.
- Resolver mapping from question/help-request/draft/session IDs to canonical student/task owners.
- Positive/negative teacher matrix covering current, stale, previous, unassigned, suspended and wrong-scope cases.

<threat_model>
Assets: student questions/conversations, assistance notes, teacher assignment state, AI drafts. Trust boundaries: teacher group to active Actor; queue/task identifiers to current ownership; draft/request IDs to student scope. Abuse cases: any teacher browses any question, previous dispatch keeps access, queue row grants profile/history, draft ID bypasses question policy, teacher role gains curriculum mutation, takeover race is accidentally redesigned. Mitigations: current task fact and explicit purpose per action, owner-resolving dependency, fresh status/assignment, separate local capability for broader/curriculum work, handler consumes resolved object, preserve Phase 475 transaction boundary. Validation: T-472-05, T-472-06, T-472-07.
</threat_model>
