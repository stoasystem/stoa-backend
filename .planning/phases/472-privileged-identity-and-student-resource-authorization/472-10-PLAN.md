# Plan 472-10: Executable route inventory, safe reconciliation, and P0 evidence gate

<objective>
Prove full registered-route coverage, reconcile existing privileged identities through dry-run-first auto-tightening only, and assemble focused P0 evidence without claiming production mutation or unrelated global-suite health.
</objective>

## Artifacts this phase produces

- `route_inventory.py`: recursive FastAPI dependency-tree inspection of executable `AuthorizationSpec` metadata plus ID detection.
- `generate_route_authorization_inventory.py` and deterministic checked `docs/security/route-authorization-inventory.json` projected into `x-stoa-authorization` OpenAPI.
- `reconcile_privileged_identities.py`: read-only default inventory/plan, checkpointed/idempotent optional apply, automatic tightening/removal only, privilege increases requiring external approved command.
- `docs/security/phase-472-evidence.md`: commands/results, old P0 reproductions, inventory hash, redaction checks, dry-run summary, sandbox/read-only placeholders with approval status, rollback, no-production-mutation statement, and honest full-suite baseline.

<threat_model>
Assets: complete route coverage, notification/event/push-token ownership, existing privileged accounts/groups/grants, migration evidence and rollback. Trust boundaries: every FastAPI router registered in `main.py` to generated inventory; production-shaped identity data to migration classification; optional apply to Cognito/local changes. Abuse cases: separately registered router is omitted, forgotten indirect/body notification identifier lacks policy, metadata exists without dependency, migration silently elevates historical tutor mismatch, a new active tutor contract hides behind a broad scan exemption, batch partially applies then duplicates, logs leak emails/tokens, evidence fabricates live success. Mitigations: recursive executable graph and synthetic route/terminology mutation tests, fail unknown/unclassified routes and identifiers, exact semantic terminology allowlist, read-only default, deterministic idempotency/checkpoints, auto-remove/suspend only, redacted summaries, deny-first rollback, explicit not-run evidence. Validation: T-472-01 through T-472-09, especially T-472-06/T-472-08/T-472-09.
</threat_model>
