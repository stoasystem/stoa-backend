# Plan 472-13: Recursive dependency identifier inventory

<objective>
Close G-03 by making identifier discovery cover the full FastAPI dependency tree and nested Pydantic/Annotated/container annotations, then fail closed when a public/global or mismatched policy route carries an undeclared sensitive identifier.
</objective>

## Artifacts this phase produces

None. This plan extends the existing route inventory and regenerates its checked JSON/OpenAPI projection; it creates no new module or public symbol.

<must_haves>
Truths: all dependency-tree and nested annotation identifiers are discovered, undeclared public/global sensitive identifiers fail, and generated projections stay identical. Artifacts: none newly created; the checked inventory is regenerated. Key link: one recursive projection feeds validation, JSON, and OpenAPI.
</must_haves>

<threat_model>
Assets: completeness of the 219-operation authorization inventory and every student/question/report/event/token identifier. Trust boundaries: Python type annotations and FastAPI `Dependant` graph to inventory classification; endpoint declarations and executable dependency policies to fail-closed validation; runtime inventory to release evidence/OpenAPI. Threats: nested `Depends` parameters disappear, `Annotated`/Optional/list/dict/model containers hide IDs, a public/global marker bypasses owner policy, a broad declaration disguises a resource read as a public command, or generated evidence remains stale. Mitigations: one cycle-safe recursive annotation walker over every dependant's path/query/body fields, canonical alias/path normalization, three separate classification rules, exact command-local declarations on deliberately public endpoints, self-only declarations for semantically valid authenticated-global identifiers, compatible executable specs for safe-public identifiers, real-route assertions, mutation canaries, and byte-identical regeneration. Verification: the 11 observed real blind spots; positive and negative cases for each public/global class; actual `/auth/register` and `/teacher-applications` declarations; and full inventory regression. Residual risk: dynamically assembled opaque payloads without declared schema cannot be statically inspected and must be explicitly unclassified/denied; no allow is inferred. Cross-phase deferrals: Phase 474 owns global CI fixtures and Phase 475 owns data-write atomicity; neither affects this route proof. No phase-owned High/Critical threat may remain unresolved.
</threat_model>
