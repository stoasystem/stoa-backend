# Plan 39-01: Metadata-only Export Backend

**Phase:** 39 - Metadata-only Export Backend
**Milestone:** v1.7 Recovery Evidence Export & Admin Credential Operations
**Status:** Planned
**Created:** 2026-06-05

## Objective

Add the backend metadata-only recovery evidence export endpoint and focused tests, preserving the v1.6/v1.7 privacy and read-only boundaries.

## Tasks

1. Add a service module for recovery evidence serialization:
   - explicit job allowlist
   - explicit target allowlist
   - explicit audit allowlist
   - redaction for private artifact text and metadata
   - response assembly with `exported_at`, `request_id`, `scope`, `complete`, `filters`, sections, `next_tokens`, and `privacy`
2. Add `GET /admin/reports/recovery-evidence` to `src/stoa/routers/admin.py`:
   - admin-only dependency
   - `job_id` exact export mode
   - recent jobs bounded mode
   - include flags and limit params
   - pagination token validation
   - request ID extraction from safe headers
3. Add tests in `tests/test_admin_report_ops.py`:
   - admin-only behavior
   - exact job export with targets and audit
   - recent jobs export
   - missing job
   - invalid target/audit/job pagination token
   - privacy denylist absence
   - read-only behavior by asserting mutation functions are not called
4. Update Phase 39 verification and summary.
5. Run:
   - `uv run pytest -q tests/test_admin_report_ops.py`
   - `uv run ruff check src/stoa/routers/admin.py src/stoa/services/report_recovery_evidence_service.py tests/test_admin_report_ops.py`
   - `git diff --check`

## Acceptance Criteria

- EXPORT-01, EXPORT-02, and EXPORT-03 are complete for the backend MVP.
- Export responses are metadata-only and explicitly allowlisted.
- Endpoint is read-only and admin-only.
- No CDK change is introduced.
- Focused tests and lint pass.
